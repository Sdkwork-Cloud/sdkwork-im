# 线上环境 PostgreSQL 数据库配置教程

本文说明 SDKWork Chat server 版本如何接入 PostgreSQL。线上环境不要使用根目录 `.env.postgres`，也不要把数据库密码写入 Git 仓库、发布包或进程启动命令。

当前应用标准：

- app code: `chat`
- public route: `/sdkwork/chat`
- package name: `sdkwork-chat`
- server database default: PostgreSQL
- desktop database default: SQLite

## 1. 标准目录

Linux server/service/container 使用：

```text
/opt/sdkwork/chat                 # archive install root
/etc/sdkwork/chat                 # config and secret root
/var/lib/sdkwork/chat             # durable data
/var/log/sdkwork/chat             # file logs
/run/sdkwork/chat                 # runtime state
```

核心配置文件：

```text
/etc/sdkwork/chat/
  chat.toml
  server.env
  postgresql.yaml
  database.secret
  redis.secret
```

Windows Service 使用 `%ProgramFiles%/sdkwork/chat` 和 `%ProgramData%/sdkwork/chat`。macOS service 使用 `/usr/lib/sdkwork/chat`、`/Library/Application Support/sdkwork/chat`、`/Library/Logs/sdkwork/chat`。

## 2. 数据库规划

建议线上为 SDKWork Chat 创建独立数据库、独立应用账号、独立 schema：

```text
database: sdkwork_chat_prod
schema: sdkwork_chat_prod
username: sdkwork_chat_prod
password: generated by password manager
```

应用账号只授予应用所需权限，不使用 PostgreSQL 超级用户。

## 3. 创建数据库和账号示例

以下 SQL 由 DBA 或部署管理员执行，按实际安全策略调整：

```sql
CREATE ROLE "sdkwork_chat_prod" LOGIN PASSWORD 'replace-with-generated-password';
CREATE DATABASE sdkwork_chat_prod OWNER "sdkwork_chat_prod";
\connect sdkwork_chat_prod
CREATE SCHEMA IF NOT EXISTS sdkwork_chat_prod AUTHORIZATION "sdkwork_chat_prod";
GRANT CONNECT ON DATABASE sdkwork_chat_prod TO "sdkwork_chat_prod";
GRANT USAGE, CREATE ON SCHEMA sdkwork_chat_prod TO "sdkwork_chat_prod";
ALTER DEFAULT PRIVILEGES IN SCHEMA sdkwork_chat_prod
  GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO "sdkwork_chat_prod";
ALTER DEFAULT PRIVILEGES IN SCHEMA sdkwork_chat_prod
  GRANT USAGE, SELECT, UPDATE ON SEQUENCES TO "sdkwork_chat_prod";
ALTER ROLE "sdkwork_chat_prod" SET search_path TO sdkwork_chat_prod, public;
```

如果数据库和 schema 已由平台提前创建，只需要确认应用账号具备读写和迁移所需权限。

## 4. chat.toml

`chat.toml` 是 server 运行配置入口：

```toml
[runtime]
environment = "production"
deployment_mode = "server"
app_code = "chat"

[server]
bind_address = "0.0.0.0:18080"
trust_forwarded_headers = true

[public_endpoints]
base_url = "https://chat.example.com/sdkwork/chat"
api_base_url = "https://chat.example.com/sdkwork/chat"
websocket_base_url = "wss://chat.example.com"
docs_base_url = "https://chat.example.com/sdkwork/chat/docs"

[paths]
config_directory = "/etc/sdkwork/chat"
config_file = "/etc/sdkwork/chat/chat.toml"
data_directory = "/var/lib/sdkwork/chat"
log_directory = "/var/log/sdkwork/chat"
cache_directory = "/var/cache/sdkwork/chat"
runtime_directory = "/run/sdkwork/chat"

[database]
engine = "postgresql"
host = "postgres.internal.example.com"
port = 5432
database = "sdkwork_chat_prod"
schema = "sdkwork_chat_prod"
username = "sdkwork_chat_prod"
password_file = "/etc/sdkwork/chat/database.secret"
ssl_mode = "require"
max_connections = 20

[redis]
enabled = true
host = "redis.internal.example.com"
port = 6379
database = 0
password_file = "/etc/sdkwork/chat/redis.secret"
key_prefix = "chat"
tls = false
max_connections = 16
```

## 5. server.env

`server.env` 维护进程级覆盖项，不保存明文数据库密码：

```env
SDKWORK_CHAT_DEPLOYMENT_MODE=server
SDKWORK_CHAT_CONFIG_FILE=/etc/sdkwork/chat/chat.toml
SDKWORK_CHAT_DATA_DIR=/var/lib/sdkwork/chat
SDKWORK_CHAT_LOG_DIR=/var/log/sdkwork/chat
SDKWORK_CHAT_RUN_DIR=/run/sdkwork/chat
SDKWORK_CHAT_SERVER_BIND=0.0.0.0:18080
SDKWORK_CHAT_SERVER_BASE_URL=https://chat.example.com/sdkwork/chat
SDKWORK_CHAT_SERVER_API_BASE_URL=https://chat.example.com/sdkwork/chat
SDKWORK_CHAT_SERVER_WEBSOCKET_BASE_URL=wss://chat.example.com
SDKWORK_CHAT_DATABASE_ENGINE=postgresql
SDKWORK_CHAT_DATABASE_HOST=postgres.internal.example.com
SDKWORK_CHAT_DATABASE_PORT=5432
SDKWORK_CHAT_DATABASE_NAME=sdkwork_chat_prod
SDKWORK_CHAT_DATABASE_SCHEMA=sdkwork_chat_prod
SDKWORK_CHAT_DATABASE_USERNAME=sdkwork_chat_prod
SDKWORK_CHAT_DATABASE_PASSWORD_FILE=/etc/sdkwork/chat/database.secret
SDKWORK_CHAT_DATABASE_SSL_MODE=require
SDKWORK_CHAT_DATABASE_MAX_CONNECTIONS=20
SDKWORK_CHAT_LOG_LEVEL=info
CRAW_CHAT_BROWSER_ORIGINS=https://chat.example.com
```

## 6. postgresql.yaml

`postgresql.yaml` 是安装和初始化辅助配置，保持拆分字段：

```yaml
provider: postgresql

connection:
  host: postgres.internal.example.com
  port: 5432
  database: sdkwork_chat_prod
  username: sdkwork_chat_prod
  passwordFile: /etc/sdkwork/chat/database.secret
  sslmode: require
  applicationName: sdkwork-chat-server
  connectTimeoutSeconds: 10

schema:
  name: sdkwork_chat_prod
  provisioningMode: none
  migrationMode: apply
  expectedVersion: latest

pool:
  minConnections: 5
  maxConnections: 20
  idleTimeoutSeconds: 300
  maxLifetimeSeconds: 1800
```

## 7. database.secret

密码文件只包含密码本身：

```text
replace-with-generated-password
```

权限建议：

```bash
sudo chown root:sdkwork /etc/sdkwork/chat/database.secret
sudo chmod 0640 /etc/sdkwork/chat/database.secret
```

Windows Service 场景下，应确保服务运行账号对 `%ProgramData%/sdkwork/chat/database.secret` 有读取权限，其他普通用户没有读取权限。

## 8. 验证配置

Windows Service 示例：

```powershell
Test-Path "$env:ProgramData\sdkwork\chat\chat.toml"
Test-Path "$env:ProgramData\sdkwork\chat\server.env"
Test-Path "$env:ProgramData\sdkwork\chat\postgresql.yaml"
Test-Path "$env:ProgramData\sdkwork\chat\database.secret"
```

Linux 示例：

```bash
test -f /etc/sdkwork/chat/chat.toml
test -f /etc/sdkwork/chat/server.env
test -f /etc/sdkwork/chat/postgresql.yaml
test -f /etc/sdkwork/chat/database.secret
```

数据库客户端验证：

```bash
PGPASSWORD="$(cat /etc/sdkwork/chat/database.secret)" \
psql "postgresql://sdkwork_chat_prod@postgres.internal.example.com:5432/sdkwork_chat_prod?sslmode=require"
```

## 9. 启动和托管

线上环境使用 server 生命周期脚本或服务管理器托管：

```bash
bash bin/install-server.sh --instance default
bash bin/init-config-server.sh --instance default
bash bin/init-storage-server.sh --instance default --mode verify-only
bash bin/verify-server.sh --instance default
bash bin/install-service-server.sh --instance default
bash bin/start-server.sh --instance default --release
```

Windows Service 使用 `install-service-server.ps1` 生成并注册 `CrawChatServer`。前台启动、systemd、launchd 和 Windows Service 都使用同一配置入口：`craw-chat-server --config <config-root>/chat.toml`。

本地命令 `pnpm dev`、`pnpm dev:postgres`、`pnpm dev:sqlite`、`pnpm tauri:dev`、`pnpm tauri:dev:postgres`、`pnpm tauri:dev:sqlite` 只用于开发。desktop 发布包默认 SQLite，不要求 PostgreSQL 首次运行。

## 10. 发布前检查

- PostgreSQL 主备、备份、恢复演练已完成。
- 应用账号不是超级用户。
- 密码文件未进入镜像、Git、发布包或日志。
- 连接池配置与 PostgreSQL `max_connections` 和实例容量匹配。
- `SDKWORK_CHAT_SERVER_BASE_URL` 使用 `/sdkwork/chat` 路由。
- `SDKWORK_CHAT_LOG_DIR` 指向 `/var/log/sdkwork/chat` 或平台等价日志目录。
- `CRAW_CHAT_BROWSER_ORIGINS` 只包含可信前端域名。
- API 和 WebSocket 使用 HTTPS/WSS。

开发环境和线上环境不要共用同一个配置文件。线上也不要直接复制 `.env.postgres` 作为服务配置。
