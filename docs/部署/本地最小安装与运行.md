### 5.1 烟雾检查脚�?

```powershell

```


`tools/smoke/local_stack_smoke.*` now sends the SDKWork AppContext projection directly to the local node for controlled local checks. It does not generate local SDKWork credentials and does not read any craw-chat public token secret. Protected service profiles should run with `CRAW_CHAT_APP_CONTEXT_REQUIRE_SIGNATURE=true` and a non-public `CRAW_CHAT_APP_CONTEXT_SIGNATURE_SECRET`; the smoke scripts read the same local runtime config or injected environment secret and add `x-sdkwork-context-signature`. The trusted gateway uses the same signing contract after it validates appbase dual tokens and rewrites the context projection.

The smoke projection uses these headers:

- `x-sdkwork-tenant-id: t_demo`
- `x-sdkwork-user-id: u_demo`
- `x-sdkwork-actor-id: u_demo`
- `x-sdkwork-actor-kind: user`
- `x-sdkwork-session-id: s_demo`
- `x-sdkwork-device-id: d_demo`
- `x-sdkwork-permission-scope: chat.write`
- `x-sdkwork-context-signature: <gateway-generated-signature>`

End-to-end smoke script:

```powershell
powershell -ExecutionPolicy Bypass -File tools\smoke\end_to_end_smoke.ps1
```

## 5.2 Authentication Notes

`sdkwork-appbase` owns login, user, tenant, organization, IAM session, dual token validation, and the authoritative IAM context. `craw-chat` consumes only the verified AppContext projection provided by the trusted edge. The local scripts no longer generate or validate craw-chat-owned JWTs.

- `bin/init-config-local.ps1` and `bin/init-config-local.sh` write `CRAW_CHAT_FRIEND_REQUEST_CURSOR_HS256_SECRET` only for friend-request cursor signing, and write `CRAW_CHAT_APP_CONTEXT_REQUIRE_SIGNATURE=true` plus `CRAW_CHAT_APP_CONTEXT_SIGNATURE_SECRET` for signed AppContext projection checks.
- `bin/start-local.*` forwards `CRAW_CHAT_FRIEND_REQUEST_CURSOR_HS256_SECRET`, `CRAW_CHAT_APP_CONTEXT_REQUIRE_SIGNATURE`, and `CRAW_CHAT_APP_CONTEXT_SIGNATURE_SECRET` when configured and does not require a public token secret.
- `tools/smoke/local_stack_smoke.*` and `tools/smoke/end_to_end_smoke.ps1` send signed `x-sdkwork-*` AppContext projection headers when a signature secret is configured.
- Production deployments must validate SDKWork auth/access tokens at the appbase/gateway boundary, then forward a signed AppContext projection to craw-chat.

For manual `curl` examples, use the same projection headers:

```bash
APP_CONTEXT_HEADERS=(
  -H "x-sdkwork-tenant-id: t_demo"
  -H "x-sdkwork-user-id: u_demo"
  -H "x-sdkwork-actor-id: u_demo"
  -H "x-sdkwork-actor-kind: user"
  -H "x-sdkwork-session-id: s_demo"
  -H "x-sdkwork-device-id: d_demo"
  -H "x-sdkwork-permission-scope: chat.write"
```
