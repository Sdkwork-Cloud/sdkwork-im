`tools/smoke/local_stack_smoke.*` now sends SDKWork dual-token request headers to the local node for controlled local checks. The auth token and access token carry tenant, user, session, actor, device, data-scope, and permission claims; the local node resolves SDKWork context from those claims instead of accepting client-controlled `x-sdkwork-*` identity projection headers. Trusted gateways may still use signed AppContext projection on service-to-service hops after they validate real appbase dual tokens.

The smoke client request uses these headers:

- `Authorization: Bearer <auth-token-jwt>`
- `Access-Token: <access-token-jwt>`

End-to-end smoke script:

```powershell
powershell -ExecutionPolicy Bypass -File tools\smoke\end_to_end_smoke.ps1
```

## 5.2 Authentication Notes

`sdkwork-appbase` owns login, user, tenant, organization, IAM session, dual token validation, and the authoritative IAM context. `sdkwork-im` consumes verified SDKWork auth/access token claims at public client entrypoints, and may consume signed AppContext projection only from trusted edge or service-to-service boundaries.

- `bin/init-config-local.ps1` and `bin/init-config-local.sh` write `sdkwork_im_FRIEND_REQUEST_CURSOR_HS256_SECRET` only for friend-request cursor signing, and write `sdkwork_im_APP_CONTEXT_REQUIRE_SIGNATURE=true` plus `sdkwork_im_APP_CONTEXT_SIGNATURE_SECRET` for signed AppContext projection checks.
- `bin/start-local.*` forwards `sdkwork_im_FRIEND_REQUEST_CURSOR_HS256_SECRET`, `sdkwork_im_APP_CONTEXT_REQUIRE_SIGNATURE`, and `sdkwork_im_APP_CONTEXT_SIGNATURE_SECRET` when configured and does not require a public token secret.
- `tools/smoke/local_stack_smoke.*` and `tools/smoke/end_to_end_smoke.ps1` send `Authorization` and `Access-Token` headers with SDKWork-shaped local JWT claim fixtures.
- Production deployments must validate SDKWork auth/access tokens at the appbase/gateway boundary before forwarding requests or deriving any signed service-to-service AppContext projection.

For manual `curl` examples, use SDKWork dual-token headers:

```bash
AUTH_TOKEN="<SDKWork auth token JWT>"
ACCESS_TOKEN="<SDKWork access token JWT>"
DUAL_TOKEN_HEADERS=(
  -H "Authorization: Bearer ${AUTH_TOKEN}"
  -H "Access-Token: ${ACCESS_TOKEN}"
)
```
