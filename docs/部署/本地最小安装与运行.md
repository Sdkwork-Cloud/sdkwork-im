# 本地最小安装与运行

本文档描述 `craw-chat` 当前阶段的 `local-minimal` 单节点运行 profile。

## 1. 适用范围

该 profile 适用于：

- 本地开发
- 接口联调
- 最小可安装演示
- 架构主链路验证

该 profile 目前以内存态实现以下能力：

- 会话创建
- 会话成员查询与增删
- 已读游标推进与未读查询
- Inbox 会话列表查询
- 消息发送
- timeline 查询
- 流式会话创建
- RTC 信令会话创建
- 媒体上传与资源绑定
- 站内通知查询
- 自动化执行与结果回查
- 审计记录导出
- 节点级运维诊断

## 2. 环境要求

- Rust stable toolchain
- 可用的 `cargo`
- 可选：Docker Desktop / Docker Engine
- 可选：`docker compose`

## 3. 启动方式

### 3.1 Bootstrap 脚本

```powershell
powershell -ExecutionPolicy Bypass -File deployments\scripts\bootstrap-local.ps1
```

该脚本会执行：

- Docker 可用性检查
- `local-minimal` compose 构建与启动
- `tools/smoke/local_stack_smoke.ps1` 烟雾检查

若 Docker daemon 不可用，脚本会在预检阶段直接失败并返回明确提示。

### 3.2 Docker Compose

```bash
docker compose -f deployments/docker-compose/local-minimal.yml up -d --build
```

### 3.3 PowerShell 本地进程

```powershell
./scripts/run-local-minimal.ps1
```

### 3.4 Bash 本地进程

```bash
./scripts/run-local-minimal.sh
```

### 3.5 手动启动

```bash
cargo run -p local-minimal-node --offline
```

如需覆盖绑定地址，可设置：

```bash
export CRAW_CHAT_BIND_ADDR=0.0.0.0:18090
```

## 4. 默认端口与适配器

- `18090` - `local-minimal-node`
- 缺省绑定地址：`127.0.0.1:18090`
- 容器推荐绑定地址：`0.0.0.0:18090`

当前 profile 使用 `adapters/local-memory` 作为默认本地适配器包：

- `MemoryCommitJournal`
- `MemoryMetadataStore`
- `MemoryTimelineProjectionStore`

该适配器包用于：

- 本地开发
- 单节点联调
- 最小安装验证
- 契约回归测试

## 5. 健康检查

```bash
curl http://127.0.0.1:18090/healthz
curl http://127.0.0.1:18090/readyz
```

### 5.1 烟雾检查脚本

```powershell
powershell -ExecutionPolicy Bypass -File tools\smoke\local_stack_smoke.ps1
```

```bash
bash tools/smoke/local_stack_smoke.sh
```

`tools/smoke/local_stack_smoke.*` 会按以下顺序解析 smoke bearer：

1. 显式传入 `-BearerToken` / `--bearer-token`
2. 显式传入 `-PublicBearerSecret` / `--public-bearer-secret`
3. `.runtime/local-minimal/config/local-minimal.env` 中的 `CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET`
4. Docker `local-minimal` profile 内置的重复执行 dev secret `local-minimal-public-dev-secret`

端到端脚本：

```powershell
powershell -ExecutionPolicy Bypass -File tools\smoke\end_to_end_smoke.ps1
```

## 5.2 认证说明

业务接口需要提供认证上下文。

- `local-minimal` public 路由要求 `Authorization: Bearer <jwt>`，并使用 `CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET` 做 HS256 验签。
- `bin/init-config-local.ps1` / `bin/init-config-local.sh` 会在 `.runtime/local-minimal/config/local-minimal.env` 中生成并写入该 secret。
- `bin/start-local.*` 与 `tools/smoke/local_stack_smoke.*` 会优先读取同一份 secret。
- `deployments/docker-compose/local-minimal.yml` 为 Docker smoke 固定注入 dev secret `local-minimal-public-dev-secret`，保证 `deploy-local` / `bootstrap-local` 可重复执行。
- 如需手工执行下文 `curl` 示例，可先生成签名 bearer：

```bash
export CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET="${CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET:-$(grep '^CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET=' .runtime/local-minimal/config/local-minimal.env 2>/dev/null | cut -d= -f2-)}"
if [ -z "$CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET" ]; then
  export CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET=local-minimal-public-dev-secret
fi

base64url() {
  openssl base64 -A | tr '+/' '-_' | tr -d '='
}

sign_local_minimal_bearer() {
  local payload_json="$1"
  local header_segment payload_segment signing_input signature_segment
  header_segment="$(printf '%s' '{"alg":"HS256","typ":"JWT"}' | base64url)"
  payload_segment="$(printf '%s' "$payload_json" | base64url)"
  signing_input="${header_segment}.${payload_segment}"
  signature_segment="$(
    printf '%s' "$signing_input" \
      | openssl dgst -binary -sha256 -hmac "$CRAW_CHAT_PUBLIC_BEARER_HS256_SECRET" \
      | base64url
  )"
  printf 'Bearer %s.%s' "$signing_input" "$signature_segment"
}

export DEMO_BEARER="$(sign_local_minimal_bearer '{"tenant_id":"t_demo","sub":"u_demo","sid":"s_demo"}')"
export OPS_AUDIT_BEARER="$(sign_local_minimal_bearer '{"tenant_id":"t_demo","sub":"u_ops_audit_demo","sid":"s_ops_audit_demo","permissions":["audit.read","ops.read"]}')"
```

- 上述示例依赖 `openssl`；若已显式提供 `--bearer-token`，则 smoke 脚本不会再生成 token。
- `DEMO_BEARER` 适用于普通业务接口。
- `OPS_AUDIT_BEARER` 适用于 `audit/export` 与 `ops/*` 这类高权限接口。
- 生产部署必须在 edge/session gateway 完成 JWT 验签、issuer/audience 校验、租户绑定与权限校验，再向内部服务透传可信上下文。

## 6. 示例调用

### 6.1 创建会话

```bash
curl -X POST http://127.0.0.1:18090/api/v1/conversations \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "conversationId":"c_demo",
    "conversationType":"group"
  }'
```

### 6.2 发送消息

```bash
curl -X POST http://127.0.0.1:18090/api/v1/conversations/c_demo/messages \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "clientMsgId":"client_demo",
    "summary":"hello",
    "text":"hello"
  }'
```

也可以直接发送结构化 `parts`：

```bash
curl -X POST http://127.0.0.1:18090/api/v1/conversations/c_demo/messages \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "clientMsgId":"client_parts_demo",
    "summary":"media message",
    "parts":[
      {
        "kind":"text",
        "text":"caption"
      },
      {
        "kind":"media",
        "mediaAssetId":"ma_demo",
        "resource":{
          "uuid":"res_demo",
          "type":"image",
          "mimeType":"image/png",
          "size":42,
          "name":"demo.png",
          "extension":"png"
        }
      }
    ]
  }'
```

群聊/agent 会话创建后，creator 会自动生成一条 `owner` 成员记录；后续消息发送会优先尝试从成员表回填 `sender.memberId`。

### 6.3 查询会话成员

```bash
curl http://127.0.0.1:18090/api/v1/conversations/c_demo/members \
  -H "Authorization: $DEMO_BEARER"

curl -X POST http://127.0.0.1:18090/api/v1/conversations/c_demo/members/add \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "principalId":"ag_demo",
    "principalKind":"agent",
    "role":"member"
  }'

curl -X POST http://127.0.0.1:18090/api/v1/conversations/c_demo/members/remove \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "memberId":"cm_c_demo_ag_demo"
  }'
```

### 6.4 查询 timeline

```bash
curl http://127.0.0.1:18090/api/v1/conversations/c_demo/messages \
  -H "Authorization: $DEMO_BEARER"
```

### 6.5 推进已读游标

```bash
curl http://127.0.0.1:18090/api/v1/conversations/c_demo/read-cursor \
  -H "Authorization: $DEMO_BEARER"

curl -X POST http://127.0.0.1:18090/api/v1/conversations/c_demo/read-cursor \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "readSeq":1,
    "lastReadMessageId":"msg_c_demo_1"
  }'
```

该接口返回成员级已读游标视图，并包含 `unreadCount`，用于本地最小 profile 下的多端同步基础验证。

### 6.6 查询会话摘要

```bash
curl http://127.0.0.1:18090/api/v1/conversations/c_demo \
  -H "Authorization: $DEMO_BEARER"
```

### 6.7 查询 Inbox

```bash
curl http://127.0.0.1:18090/api/v1/inbox \
  -H "Authorization: $DEMO_BEARER"
```

该接口返回当前认证主体具备活跃成员关系的会话列表，并附带 `unreadCount`、`lastSummary`、`conversationType` 等摘要字段。

### 6.8 打开流

```bash
curl -X POST http://127.0.0.1:18090/api/v1/streams \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "streamId":"st_demo",
    "streamType":"custom.delta.text",
    "scopeKind":"conversation",
    "scopeId":"c_demo",
    "durabilityClass":"durableSession",
    "schemaRef":"custom.delta.text.v1"
  }'
```

### 6.9 创建 RTC 信令会话

```bash
curl -X POST http://127.0.0.1:18090/api/v1/rtc/sessions \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "rtcSessionId":"rtc_demo",
    "conversationId":"c_demo",
    "rtcMode":"voice"
  }'
```

当会话带有 `conversationId` 时，后续 `invite/accept/reject/end` 会自动写入该会话的时间线，消息摘要分别为 `rtc.invite`、`rtc.accept`、`rtc.reject`、`rtc.end`。

### 6.10 创建媒体上传

```bash
curl -X POST http://127.0.0.1:18090/api/v1/media/uploads \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "mediaAssetId":"ma_demo",
    "resource":{
      "uuid":"res_demo",
      "type":"image",
      "mimeType":"image/png",
      "size":42,
      "name":"demo.png",
      "extension":"png",
      "metadata":{"origin":"local-minimal"},
      "prompt":"poster"
    }
  }'
```

### 6.11 完成媒体上传并查询资源

```bash
curl -X POST http://127.0.0.1:18090/api/v1/media/uploads/ma_demo/complete \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "bucket":"local-media",
    "objectKey":"tenant/t_demo/ma_demo/demo.png",
    "storageProvider":"local",
    "url":"https://cdn.example.com/ma_demo/demo.png",
    "checksum":"sha256:demo"
  }'

curl http://127.0.0.1:18090/api/v1/media/ma_demo \
  -H "Authorization: $DEMO_BEARER"
```

### 6.12 把媒体资源绑定到消息

```bash
curl -X POST http://127.0.0.1:18090/api/v1/media/ma_demo/attach \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "conversationId":"c_demo",
    "clientMsgId":"client_attach",
    "summary":"poster asset",
    "text":"see attachment"
  }'
```

### 6.13 查询通知

消息提交后，`local-minimal-node` 会通过编排层触发 `notification.requested -> notification.dispatched` 最小流水。

```bash
curl http://127.0.0.1:18090/api/v1/notifications \
  -H "Authorization: $DEMO_BEARER"
```

### 6.14 触发自动化执行

```bash
curl -X POST http://127.0.0.1:18090/api/v1/automation/executions \
  -H "Authorization: $DEMO_BEARER" \
  -H "Content-Type: application/json" \
  -d '{
    "executionId":"ae_demo",
    "triggerType":"webhook.manual",
    "targetKind":"workflow",
    "targetRef":"wf_demo",
    "inputPayload":"{\"conversationId\":\"c_demo\"}"
  }'
```

### 6.15 导出审计记录

```bash
curl http://127.0.0.1:18090/api/v1/audit/export \
  -H "Authorization: $OPS_AUDIT_BEARER"
```

### 6.16 查询运维诊断

```bash
curl http://127.0.0.1:18090/api/v1/ops/health \
  -H "Authorization: $OPS_AUDIT_BEARER"

curl http://127.0.0.1:18090/api/v1/ops/cluster \
  -H "Authorization: $OPS_AUDIT_BEARER"

curl http://127.0.0.1:18090/api/v1/ops/diagnostics \
  -H "Authorization: $OPS_AUDIT_BEARER"
```

## 7. 当前限制

- 当前 profile 为单进程内存态实现
- 没有外部数据库和消息队列
- 重启后状态不会保留
- 主要用于最小闭环验证，不代表最终生产部署形态
