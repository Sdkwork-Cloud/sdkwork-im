# 生命周期脚本与运行目录标准

## 1. 目标

本标准定义 `craw-chat` 本地最小可运行版本的安装、初始化配置、启动、状态检查、重启、停止脚本，以及统一运行目录布局，确保：

- 首次下载后可快速安装和启动
- 脚本在 Windows、Linux/macOS 上职责一致
- 运行目录、配置、日志、PID 文件位置稳定

## 2. 运行目录

统一运行目录：

- `.runtime/local-minimal/config`
- `.runtime/local-minimal/logs`
- `.runtime/local-minimal/pids`

关键文件：

- `.runtime/local-minimal/config/local-minimal.env`
- `.runtime/local-minimal/logs/local-minimal-node.out.log`
- `.runtime/local-minimal/logs/local-minimal-node.err.log`
- `.runtime/local-minimal/pids/local-minimal-node.pid`

## 3. 脚本职责

### 3.1 安装

- `bin/install-local.ps1`
- `bin/install-local.sh`
- `bin/install-local.cmd`

职责：

- 检查 `cargo` 可用
- 初始化 `.runtime` 目录
- 调用配置初始化脚本
- 执行 `cargo build --offline`

### 3.2 初始化配置

- `bin/init-config-local.ps1`
- `bin/init-config-local.sh`
- `bin/init-config-local.cmd`

职责：

- 生成或更新 `local-minimal.env`
- 写入 `CRAW_CHAT_BIND_ADDR`

### 3.3 启动

- `bin/start-local.ps1`
- `bin/start-local.sh`
- `bin/start-local.cmd`

职责：

- 先调用安装脚本保证二进制可用
- 检查 PID 文件，避免重复启动
- 启动 `local-minimal-node`
- 轮询 `/healthz`，健康后返回成功

### 3.4 状态

- `bin/status-local.ps1`
- `bin/status-local.sh`
- `bin/status-local.cmd`

职责：

- 展示配置文件路径
- 展示绑定地址
- 展示日志路径
- 展示运行状态和 PID

### 3.5 重启

- `bin/restart-local.ps1`
- `bin/restart-local.sh`
- `bin/restart-local.cmd`

职责：

- 顺序执行 stop + start

### 3.6 停止

- `bin/stop-local.ps1`
- `bin/stop-local.sh`
- `bin/stop-local.cmd`

职责：

- 根据 PID 文件结束后台进程
- 清理陈旧 PID 文件

## 4. 平台实现差异

### 4.1 Windows

Windows 当前采用：

- 直接 `Start-Process` 启动 `local-minimal-node.exe`
- 使用 `Invoke-WebRequest -UseBasicParsing` 做健康检查

约束说明：

- Windows PowerShell 下，`Start-Process` 对当前 Rust 可执行文件使用标准输出/错误重定向会导致后台启动不稳定
- 因此当前 Windows 后台模式优先保证“成功启动和可停止”，而不是强制捕获子进程控制台输出
- 若需要实时观察控制台输出，使用 `bin/start-local.ps1 -Foreground`

### 4.2 Linux/macOS

Unix 脚本当前采用：

- `nohup`
- 环境变量注入
- 标准输出/错误追加到日志文件

## 5. 健康检查标准

健康检查地址由 `CRAW_CHAT_BIND_ADDR` 推导，统一使用：

- `http://<resolved-host>:<port>/healthz`

特殊规则：

- 若绑定地址为 `0.0.0.0`、`::` 或 `[::]`，健康检查主机回退到 `127.0.0.1`
- 最长等待 30 秒
- 健康检查失败必须返回非零退出码

## 6. PID 文件标准

PID 文件仅允许记录一个后台进程 PID。

脚本必须满足：

- PID 文件不存在时视为未运行
- PID 文件为空或 PID 不存在时必须自动清理陈旧文件
- 重复启动时必须显式失败，而不是覆盖现有 PID

## 7. 最低回归要求

至少覆盖以下场景：

- `install-local.ps1` 可完成目录和配置初始化
- `start-local.ps1` 可在同一 PowerShell 会话内启动并通过健康检查
- `status-local.ps1` 能读取到运行状态和 PID
- `restart-local.ps1` 能完成 stop + start
- `stop-local.ps1` 能结束进程并清理 PID 文件
- `.cmd` 包装脚本能正确委派到对应 PowerShell 脚本

## 8. 建议运维用法

### 8.1 Windows

首次安装：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\bin\install-local.ps1
```

启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\bin\start-local.ps1
```

前台启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\bin\start-local.ps1 -Foreground
```

### 8.2 Linux/macOS

```bash
bash bin/install-local.sh
bash bin/start-local.sh
```
