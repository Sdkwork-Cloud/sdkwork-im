# 本地最小部署快速启动脚本

当前目录已经提供统一的 `bin/` 入口，目标是把安装、Docker 部署、本地进程启动三类操作收敛成同名跨平台脚本，避免继续直接记忆底层 `cargo` 或 `deployments/scripts` 命令。

## 1. 脚本清单

- Windows PowerShell
  - `bin/install-local.ps1`
  - `bin/deploy-local.ps1`
  - `bin/start-local.ps1`
- Windows CMD 包装
  - `bin/install-local.cmd`
  - `bin/deploy-local.cmd`
  - `bin/start-local.cmd`
- Linux / macOS
  - `bin/install-local.sh`
  - `bin/deploy-local.sh`
  - `bin/start-local.sh`

## 2. 职责边界

- `install-local`
  - 检查 `cargo`
  - 创建 `.runtime/local-minimal/logs` 与 `.runtime/local-minimal/pids`
  - 离线构建 `local-minimal-node`
- `deploy-local`
  - 用于 Docker 本地部署
  - Windows 直接委托 `deployments/scripts/bootstrap-local.ps1`
  - Linux/macOS 直接执行 `docker compose -f deployments/docker-compose/local-minimal.yml up -d --build`
- `start-local`
  - 用于本地进程直接启动
  - 自动先执行 `install-local`
  - 写出日志文件与 PID 文件
  - 默认后台启动，可切换前台模式

## 3. 推荐用法

### 3.1 Windows

安装并准备运行目录：

```cmd
bin\install-local.cmd
```

Docker 部署：

```cmd
bin\deploy-local.cmd
```

本地后台启动：

```cmd
bin\start-local.cmd
```

本地前台启动：

```cmd
powershell -ExecutionPolicy Bypass -File bin\start-local.ps1 -Foreground
```

### 3.2 Linux / macOS

首次执行前建议赋予可执行权限：

```bash
chmod +x bin/*.sh
```

安装并准备运行目录：

```bash
bash bin/install-local.sh
```

Docker 部署：

```bash
bash bin/deploy-local.sh
```

本地后台启动：

```bash
bash bin/start-local.sh
```

本地前台启动：

```bash
bash bin/start-local.sh --foreground
```

## 4. 常用参数

- `install-local`
  - `--release` / `-Release`
- `deploy-local`
  - `--skip-smoke` / `-SkipSmoke`
- `start-local`
  - `--release` / `-Release`
  - `--foreground` / `-Foreground`
  - `--bind-addr <host:port>` / `-BindAddress <host:port>`

## 5. 运行产物

脚本会在项目根目录生成以下运行时产物：

- `.runtime/local-minimal/logs/local-minimal-node.out.log`
- `.runtime/local-minimal/logs/local-minimal-node.err.log`
- `.runtime/local-minimal/pids/local-minimal-node.pid`

## 6. 关闭后台进程

Windows：

```powershell
Stop-Process -Id (Get-Content .runtime\local-minimal\pids\local-minimal-node.pid)
```

Linux / macOS：

```bash
kill "$(cat .runtime/local-minimal/pids/local-minimal-node.pid)"
```

## 7. 设计约束

- `bin/` 只是统一入口，不复制已有编排逻辑
- Docker 部署继续复用既有 `bootstrap-local.ps1` 和 compose profile
- 本地进程启动继续以 `local-minimal-node` 为唯一运行目标
- 运行目录统一落在 `.runtime/`，便于后续演进到私有化安装包
