# 安装启动配置覆盖语义标准-2026-04-05

## 1. 目标

统一 `bin/install-local.*`、`bin/start-local.*`、`bin/restart-local.*` 在 Windows PowerShell、Windows CMD 包装层、Linux/macOS Bash 三套入口上的配置行为，避免默认参数在已有配置场景下误覆盖 `local-minimal.env`。

本标准约束本地最小可运行版本，但语义同样适用于后续私有化部署、systemd/launchd/Windows Service 包装层。

## 2. 核心规则

### 2.1 显式传入绑定地址才允许覆盖配置

- 当调用方显式传入 `BindAddress` 或 `--bind-addr` 时，安装脚本必须将该地址写回 `.runtime/local-minimal/config/local-minimal.env`。
- 该覆盖行为必须等价于对 `init-config-local.*` 传递强制覆盖参数：
  - PowerShell: `-Force`
  - Bash: `--force`

### 2.2 未显式传入绑定地址时必须保留已有配置

- 当调用方没有传入绑定地址时：
  - `install-local.*` 只能确保配置文件存在。
  - 如果配置文件已经存在，脚本不得使用默认值 `127.0.0.1:18090` 重写它。
- `start-local.*` 在未显式传入绑定地址时，必须优先读取现有 `local-minimal.env` 中的 `CRAW_CHAT_BIND_ADDR`。
- 如果配置文件不存在，才允许回落到默认值 `127.0.0.1:18090`。

### 2.3 restart 只做透传，不得注入伪默认值

- `restart-local.*` 只能把用户实际提供的参数透传给 `start-local.*`。
- 如果用户没有显式传入绑定地址，`restart-local.*` 不得补一个默认地址后再传下去。

## 3. 各入口的标准行为

### 3.1 `install-local.ps1`

- 使用 `$PSBoundParameters.ContainsKey('BindAddress')` 判断用户是否显式提供了 `BindAddress`。
- 仅在显式提供时，对 `init-config-local.ps1` 透传 `-Force`。

### 3.2 `start-local.ps1`

- 不得无条件向 `install-local.ps1` 传递默认 `BindAddress`。
- 仅当调用者显式传入 `BindAddress` 时，才将其继续透传到 `install-local.ps1`。
- 启动实际进程时，解析优先级必须是：
  1. 显式传入的 `BindAddress`
  2. 配置文件中的 `CRAW_CHAT_BIND_ADDR`
  3. 默认值 `127.0.0.1:18090`

### 3.3 `restart-local.ps1`

- 仅透传调用方显式提供的 `BindAddress`。
- 不得以 `$null` 或默认值形式把绑定地址强行绑定给 `start-local.ps1`。

### 3.4 `install-local.sh`

- 必须维护 `bind_addr_provided` 之类的显式参数标记。
- 只有在 `bind_addr_provided=1` 时，才允许向 `init-config-local.sh` 追加 `--force`。

### 3.5 `start-local.sh`

- 不得把默认值预先写入 `bind_addr` 后再无条件透传给 `install-local.sh`。
- 仅在显式传入 `--bind-addr` 时，才透传该参数给安装脚本。
- 未显式传参时，必须从 `local-minimal.env` 读取 `CRAW_CHAT_BIND_ADDR`，不存在时再回落默认值。

### 3.6 `*.cmd`

- CMD 只是 PowerShell 包装层。
- `/bindAddress`、`/force` 等 Windows 风格参数规范化后，必须继承 PowerShell 同样的显式覆盖语义。

## 4. 回归用例

以下行为必须长期保留在自动化测试中：

1. `install-local.cmd /bindAddress 127.0.0.1:18111` 在已有配置文件存在时，必须把配置重写为 `127.0.0.1:18111`。
2. `start-local.ps1` 在已有 `CRAW_CHAT_BIND_ADDR=127.0.0.1:18120` 且未显式传 `BindAddress` 时，必须按 `127.0.0.1:18120` 启动，并且不得把配置回写成 `127.0.0.1:18090`。
3. `start-local.sh` 与 `install-local.sh` 必须包含与 PowerShell 版本等价的显式参数检测和强制覆盖逻辑。

## 5. 风险边界

- 当前标准仅定义“是否覆盖配置”的语义，不负责绑定地址格式合法性校验。
- 若后续引入 `systemd`、`launchd`、Windows Service 包装器，必须复用同一套语义，不得在更外层偷偷注入默认地址。
- 如果未来把端口、监听协议、证书路径拆成多字段配置，仍需遵守同样原则：
  - 显式传入才覆盖
  - 未显式传入不破坏已有配置
