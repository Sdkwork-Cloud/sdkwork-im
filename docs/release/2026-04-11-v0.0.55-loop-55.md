# v0.0.55 - Loop 55 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“只有同步 dispatch / 没有失败持久化”推进到最小可审阅的 `durable pending backlog + operator repair` 闭环

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - `SocialControlState` 新增 `pending_shared_channel_sync_requests`
  - dispatch 逻辑当前会在 trigger 未配置或 dispatch 失败时，把 ready-pair sync 请求 durable 记入 backlog
  - 新增 `POST /api/v1/control/social/runtime/repair-shared-channel-sync`
  - `repair-derived-snapshot` 与 `repair-social-runtime-dir` 当前 replay journal 时会保留 backlog
  - `SocialAggregateCountsResponse` 当前显式暴露 `pendingSharedChannelSyncRequests`
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 新增 dual-service e2e，证明 shared-channel sync 初次失败后 backlog 会落到 `social-state.json`
  - repair route 切换到真实 public HTTP trigger 后可重放 backlog，并让远端 runtime materialize linked member
