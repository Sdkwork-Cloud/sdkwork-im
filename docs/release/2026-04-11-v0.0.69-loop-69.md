# v0.0.69 - Loop 69 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“inventory 已返回 `leaseStatus`，但 operator 仍需自己推断当前 actor 能否 takeover”推进到“inventory 直接返回 current-operator `takeoverEligible`”的最小 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - `SocialSharedChannelSyncInventoryItemResponse` 新增 `takeoverEligible`
  - `PendingSharedChannelSyncRequest` 新增 `takeover_eligible_for(actor_id, actor_kind, now)`
  - pending/dead-letter inventory 当前按当前 actor 与 `control.write` 权限返回 `takeoverEligible`
  - `untracked` foreign claim 当前会显式返回 `takeoverEligible = true`，因为既有 takeover route 仍允许 legacy no-lease claim 被接管
  - 本轮没有改动 claim / release / takeover 的执行语义
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - release lifecycle 当前新增 `takeoverEligible = false -> false -> false`
  - takeover lifecycle 当前新增 `takeoverEligible = false -> true -> true -> false`
  - takeover 可见性当前同时覆盖 active foreign、legacy untracked、stale foreign 与 takeover 后 self-owned 四个状态
