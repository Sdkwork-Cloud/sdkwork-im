## 风险与未闭环
- 当前 requeue 只支持“全量 dead-letter 回灌”，没有单条/筛选 contract
- 当前 requeue 保留 `failureCount / lastError`，因此坏环境下请求会很快重新回到 dead-letter
- 当前没有 automatic retry scheduler
- 当前没有 remote republish ownership / lease / SLA，也没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth

## 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop58 已把 `shared-channel sync` 补齐到 `durable pending backlog + operator repair + next healthy ready-pair write auto-retry + repeated-failure dead-letter + dead-letter requeue`
- 下一主缺口收敛为 `automatic retry scheduler / remote republish / delivery ownership semantics`
