# v0.0.63 - Loop 63 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“operator 已可 claim pending request，但 claim 无 release lifecycle”推进到“owner-only targeted release + reclaim”的最小可审阅 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - 新增 `POST /api/v1/control/social/runtime/release-pending-shared-channel-sync-targeted`
  - targeted release 当前只允许 owner operator 清空 selected pending request 的 owner 元数据
  - foreign-owned request 的 release 当前返回 `409 shared_channel_sync_owner_conflict`
  - released request 当前返回 unowned pool，可被其他 operator 再次 claim，并复用既有 targeted republish
  - 现有 `repair-shared-channel-sync` 全量 route 保持不变
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 新增 dual-service release lifecycle e2e，证明 `claim -> foreign release 409 -> owner release -> foreign reclaim -> republish`
  - Loop62 ownership e2e 与 Loop61 targeted republish 回归继续通过

## 风险与未闭环
- 当前还没有 `claimedAt / leaseExpiresAt / SLA` 元数据
- 当前 foreign operator 仍无法对 abandoned claim 做 force release / takeover
- 当前 `repair-shared-channel-sync` 仍是粗粒度全量 operator route，尚未并入 claim lifecycle / lease policy language
- 当前没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth

## 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop63 已把 `shared-channel sync` 补齐到 `durable pending backlog + operator repair + pending inventory + targeted pending republish + pending targeted claim + republish ownership guard + pending targeted release + next healthy ready-pair write auto-retry + repeated-failure dead-letter + dead-letter inventory + dead-letter targeted/all requeue + failure-budget rearm`
- 下一主缺口收敛为 `stale-claim / lease / SLA semantics`
