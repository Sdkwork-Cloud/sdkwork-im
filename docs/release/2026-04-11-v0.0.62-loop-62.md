# v0.0.62 - Loop 62 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“operator 可按 `requestKey` 定点 republish，但缺少 owner 边界”推进到“operator 必须先 claim，再按 owner 边界 targeted republish”的最小可审阅 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - 新增 `POST /api/v1/control/social/runtime/claim-pending-shared-channel-sync-targeted`
  - pending/dead-letter inventory item 当前显式返回 `ownerActorId / ownerActorKind`
  - durable `PendingSharedChannelSyncRequest` 当前保留可选 owner 元数据
  - `claim_pending_shared_channel_sync_targeted(...)` 当前按当前 control operator 的 `actorId / actorKind` claim 被选中的 pending request
  - `republish_pending_shared_channel_sync_targeted(...)` 当前只允许 owner operator 投递已 claim request；foreign-owned 或 unclaimed request 返回 `409 shared_channel_sync_owner_conflict`
  - 现有 `repair-shared-channel-sync` 全量 route 保持不变
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 新增 dual-service ownership e2e，证明 `claim -> foreign operator 409 -> owner republish`
  - Loop61 的 targeted republish 回归升级为“先 claim 再 republish”的新合同
