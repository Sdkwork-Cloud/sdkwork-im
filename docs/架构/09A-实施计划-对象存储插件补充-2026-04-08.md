# 09A 实施计划补充：对象存储插件补全

## 目标

- 补齐对象存储 provider 从契约到运行时的最小可交付闭环。
- 复用 RTC provider 已验证过的模式，避免为 media 单独发明一套 provider 机制。

## 交付范围

1. `provider registry`
   - 新增 `deployment_profile` 选择层，顺序固定为：
     - tenant override
     - deployment_profile
     - global default
2. `adapter`
   - 新增 `object-storage-s3` 基线 crate。
   - 覆盖 `阿里云 / 腾讯云 / 火山引擎 / AWS / Google / Microsoft`。
3. `runtime`
   - `media-service` 内置 provider map。
   - 默认 deployment profile 选择 `object-storage-volcengine`。
4. `http surface`
   - `GET /api/v1/media/provider-health`
   - `GET /api/v1/media/{media_asset_id}/download-url`
5. `verification`
   - adapter contract
   - standalone service
   - local-minimal assembled runtime

## 已完成状态

- 已实现 `object-storage-s3` baseline。
- 已实现 media runtime provider 绑定。
- 已实现 canonical download URL 签发。
- 已实现 local-minimal mirror surface。
