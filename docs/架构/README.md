# Craw Chat 架构文档索引

本文档集用于冻结 `craw-chat` 的产品定义、架构标准、模块规划、技术选型、数据结构、网关协议、缓存策略、部署方案与实施计划。

目标是建设一套可同时支撑 `SaaS` 与 `私有化` 的新一代即时通信服务端，具备以下能力：

- 高并发长连接
- 分布式会话路由与在线扩缩容
- 多租户与强安全
- 标准 IM
- 通知与事件分发
- 用户可定义的数据流
- RTC 实时语音/视频信令
- 流式消息收发
- 文件资源标准化引用
- Bot / Webhook / Workflow 扩展
- 面向 AI 时代的消息内容与资源承载能力

## 冻结结论

当前版本的核心冻结结论如下：

- 采用 `统一产品，分离运行时` 的总体形态。
- `SaaS` 与 `私有化` 共享一套消息语义、事件模型、数据结构和协议契约。
- `控制面 / 数据面 / 运维面` 三层分离。
- `单会话单主写`，会话内严格有序。
- `提交日志` 与 `查询投影` 分离。
- `流` 是原生能力，不是 AI 专用能力。
- `MediaResource` 仅作为消息体中的文件资源结构使用。
- 所有基础设施通过 `Ports and Adapters` 可插拔接入。
- 当前已落地 `local-minimal + local-memory adapters` 作为最小可安装闭环。

## 实施原则

- 先冻结文档，再按文档驱动实现。
- 先实现功能正确性与架构边界，再做性能压榨。
- 先实现可观测、可回放、可恢复，再追求组件数量最少。
- 先保留先进默认栈，再通过适配器逐步替换外部依赖。
