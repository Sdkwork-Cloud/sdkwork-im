# 152CJ Loop70补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“legacy `untracked` claim 仍会被普通 takeover 路径隐式兼容”推进到“legacy `untracked` claim 必须走显式 override”的最小 contract

## 2. As-Built 事实
- inventory 当前新增 `legacyTakeoverRequired`
- `takeoverEligible` 当前只代表“普通 takeover route 可直接执行”的资格，不再把 foreign `untracked` claim 算进去
- targeted takeover request 当前新增 `allowLegacyUntracked`
- 默认 takeover 当前会拒绝 foreign `untracked` claim，并返回 `shared_channel_sync_legacy_takeover_override_required`
- 显式 `allowLegacyUntracked = true` 当前仍会复用既有 legacy compatibility path 完成 takeover
- takeover response 当前新增 `legacyOverrideUsed`，用于显式报告是否走了 legacy override

## 3. 明确边界
- 本轮没有：
  - dead-letter inventory 专项 parity 断言
  - 单独的 legacy cleanup / migration workflow
  - configurable lease window / SLA
  - timeout reclaim / automatic stale detection
  - stale-aware bulk `repair-shared-channel-sync`
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 内部的 legacy contract hardening seam，而不是完整的 governance-approved operator policy engine
