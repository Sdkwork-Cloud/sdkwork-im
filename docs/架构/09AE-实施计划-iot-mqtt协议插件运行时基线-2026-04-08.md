# 09AE - 实施计划 - iot-mqtt 协议插件运行时基线

## 目标

让 IoT provider/plugin 体系第一次拥有真实 protocol adapter crate，而不是继续停留在 registry 矩阵和架构文档。

## 最小实施面

1. 先写红测，冻结 `adapters/iot-mqtt` 与 step/架构/review 证据
2. 新增 `adapters/iot-mqtt`
3. 真实实现 `IotProtocolAdapter`：
   - `protocol_key`
   - `decode_uplink`
   - `encode_downlink`
   - `provider_health_snapshot`
4. 新增 adapter contract test
5. 回写 `docs/step`、`docs/架构`、`docs/review`
6. 在文档中显式冻结统一主链语义：
   - uplink -> `device.telemetry`
   - downlink -> `device.command`

## 约束

- 不伪造完整 IoT runtime
- 不伪造 `DeviceAccessProvider` 已接入
- 先冻结协议 adapter baseline，再继续推进接入与设备管理主链

## 放行标准

- workspace manifest 包含 `adapters/iot-mqtt`
- `adapters/iot-mqtt` 可编译且 contract test 通过
- step / 架构 / review 已显式说明这是 IoT 第一条真实 adapter 基线
- step / 架构 / review 已显式说明 `device.telemetry` 与 `device.command` 是统一主链表达
