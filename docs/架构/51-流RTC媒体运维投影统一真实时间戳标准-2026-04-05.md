# 51-流RTC媒体运维投影统一真实时间戳标准-2026-04-05

## 1. 问题背景

在完成通知、自动化、审计、会话网关、会话内核的真实时间戳整改之后，剩余服务仍存在把示例时间直接写入生产对象的问题，主要分布在以下边界：

- `streaming-service`
- `rtc-signaling-service`
- `media-service`
- `ops-service`
- `session-gateway` 的 presence 与 cluster route 归属路径
- `projection-service` 的设备注册路径

这些固定时间戳会直接破坏以下能力：

- 流式帧、RTC 信令、媒体完成事件的时序可信度
- 诊断导出与运维快照的观测可信度
- 多设备会话恢复、presence 心跳、断开事件的先后顺序
- 路由归属迁移与设备注册基线时间

## 2. 统一标准

所有由服务端在运行时生成的时间字段，必须满足以下约束：

1. 禁止在生产路径写入固定 RFC3339 字符串作为当前时间。
2. 必须统一使用共享时间源 `im_time::utc_now_rfc3339_millis()`。
3. 同一次领域动作只能生成一次时间，后续衍生对象必须复用这一次时间。
4. 时间字段必须保持 RFC3339 UTC 毫秒精度，保证字符串字典序与时间序一致。

## 3. 本轮覆盖边界

### 3.1 streaming-service

- `StreamSession.openedAt` 改为真实时间
- `StreamFrame.occurredAt` 改为真实时间
- `StreamSession.closedAt` 在 `complete` / `abort` 时改为真实时间

### 3.2 rtc-signaling-service

- `RtcSession.startedAt` 改为真实时间
- `RtcSignalEvent.occurredAt` 改为真实时间
- `RtcSession.endedAt` 在 `reject` / `end` 时改为真实时间

### 3.3 media-service

- `MediaAsset.createdAt` 改为真实时间
- `MediaAsset.completedAt` 改为真实时间
- `media.asset.created` 事件的 `occurredAt` / `committedAt` 复用 `completedAt`

### 3.4 ops-service

- `DiagnosticBundle.generatedAt` 改为每次生成时实时计算

### 3.5 session-gateway

- `SessionResumeView.resumedAt` 改为真实时间
- Presence `lastResumeAt` / `lastSeenAt` 改为真实时间
- `RealtimeDeviceRoute.boundAt` 在绑定与迁移后改为真实时间

### 3.6 projection-service

- `RegisteredDeviceView.registeredAt` 改为设备注册时实时生成

## 4. 回归测试要求

本标准要求以下测试必须存在并持续通过：

- 流式会话在打开、追加帧、关闭之间时间严格前进
- RTC 会话在建会、发信令、结束之间时间严格前进
- 媒体资源在创建、完成、事件写入之间时间严格前进
- 运维诊断包连续生成时 `generatedAt` 严格前进
- Session resume / heartbeat / disconnect 的 presence 时间严格前进
- Cluster route bind / migrate 的 `boundAt` 严格前进
- Projection device register 的 `registeredAt` 严格前进

## 5. 实施结果

本轮整改后，`services/` 与 `crates/` 中残留的固定时间字符串仅存在于：

- 测试夹具
- 序列化模型契约示例
- 测试模块内部构造数据

生产代码路径中，不再允许依赖固定时间常量充当当前时间。

## 6. 下一轮 Review 重点

时间戳问题收口后，下一轮 review 重点转向：

1. 列表排序与窗口裁剪边界是否完全依赖真实业务时间或单调序列，而不是插入顺序。
2. `streaming-service` / `rtc-signaling-service` / `projection-service` 的幂等、冲突、生命周期终态是否还存在业务边界漏洞。
3. 控制面、运维、投影、同步视图之间是否存在“状态已更新但派生视图未同步”的一致性缺口。
