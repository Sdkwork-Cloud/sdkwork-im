## 1. 本轮唯一主缺口
- Loop48 固定只做 `retention enforcement runtime truth`
- `thread subscription / notification`、`shared fanout / rebuild`、`archive / restore / retention owner` 均保持 deferred，不与本轮并行混写

## 2. 本轮交付
- 新增 `retention_policy_ref -> retention_class` runtime 派生规则，默认取 policy ref 的最后一段
- `conversation.policy_applied` 的 envelope retention class 现在跟随已归一化 policy，而不再固定为 `standard`
- policy 生效后，后续 `conversation/message mutation` commit 会继承当前 conversation policy 派生出的 retention class
- recovery replay 后继续发消息时，仍保持同一 retention truth

## 3. 运行时边界
- 本轮 retention truth 只闭合 `runtime commit metadata propagation`，不是 `archive / restore / legal-hold` 平台能力
- 派生规则当前固定为 `tenant.compliance -> compliance` 这类“取最后一段”的最小语义
- 未设置 policy 时仍回退 `standard`
