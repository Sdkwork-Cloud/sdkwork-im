## 2. 已兑现事实
- `ConversationScenario::Thread` 已进入 `im-domain-core::conversation`。
- `conversation-runtime` 新增 `CreateThreadConversationCommand` 与 dedicated `create_thread_conversation*` runtime path。
- thread create 会校验：
  - parent conversation 存在且为 `group`
  - `rootMessageId` 属于该 parent conversation
  - 创建者是 parent conversation 的 active member
  - 同一 `rootMessageId` 不会重复绑定第二个 thread conversation
- thread 现在会把 `businessType = thread / businessId = rootMessageId` 记录到 binding truth，并把 `parentConversationId / rootMessageId / threadRole` 写入 owner member attributes。
- HTTP 已新增 `POST /im/v3/api/chat/conversations/threads`，binding query 可直接读回 thread truth。
- fresh 验证：
