## 本轮实际完成

### 1. 为 automation 域补齐 tool call 生命周期模型
- 在 `im-domain-core::automation` 新增：
  - `AgentToolCallState`
  - `AgentToolCall`
- 现在工具调用生命周期具备最小结构化状态：
  - `requested`
  - `completed`
  - `failed`

### 2. 在 automation-service 内落地 Agent 响应流运行时
- `AutomationRuntime` 新增 Agent 响应流运行时状态：
  - `start_agent_response(...)`
  - `append_agent_response_delta(...)`
  - `complete_agent_response(...)`
- 响应流直接复用 `im_domain_core::stream::{StreamSession, StreamFrame}`，没有新造 Agent 私有流模型。
- Agent delta frame 的 `sender` 直接来自 `AgentSubject::sender(...)`，确保 `kind = "agent"`、`id = agent_id`、`member_id / session_id / metadata` 一并进入统一实时帧。

### 3. 在 automation-service 内落地 tool call 生命周期
- `AutomationRuntime` 新增：
  - `request_agent_tool_call(...)`
  - `complete_agent_tool_call(...)`
- 工具调用现在与 execution 作用域绑定，并复用本轮前面已经启动的 Agent 响应流上下文。

### 4. 将 Agent 生命周期事件纳入 commit journal
- 本轮新增可验证事件：
  - `automation.agent_response_started`
  - `automation.agent_response_delta`
  - `automation.agent_tool_call_requested`
  - `automation.agent_tool_call_completed`
  - `automation.agent_response_completed`
- 为避免与 execution 事件排序冲突，`AutomationRuntime` 新增按 execution 作用域递增的 `event_orders`。
