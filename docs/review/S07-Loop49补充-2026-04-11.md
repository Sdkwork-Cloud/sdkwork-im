## 2. 本轮交付
- `conversation-runtime::create_thread_conversation*` 在创建者之外，会把当前仍是 parent active member 的 root message author 一并 materialize 为 thread active member
- auto-subscribed root author 会保留 `parentConversationId / rootMessageId / threadRole = root_author` metadata，并初始化默认 read cursor
- recovery replay 后，root author 仍能在 thread 中发消息
- 现有 `notification-service::request_message_posted_notifications(...)` 无需改代码，即可继续消费 projection 中的 thread active principal 集合

## 3. 运行时边界

- 没有引入独立 `im_thread_subscription` durable model
- 没有实现 per-user unread / notification level / follow-unfollow 配置
- 没有把所有 parent members 自动拉入 thread
