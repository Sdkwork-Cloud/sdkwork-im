## 本轮实际完成

### 1. 收紧 `device` scope 的 gateway 边界
- `services/streaming-service/src/lib.rs`
  - 独立 `streaming-service` 现在拒绝直接打开或访问 `scopeKind = "device"` 的 stream。
  - `device-bound streams must be opened/accessed through an authorizing IM gateway` 已成为真实运行时约束。
- 这避免了设备路径绕过 IM gateway 的认证、设备绑定和 capability gate。

### 2. 在 `local-minimal-node` 落地设备 telemetry / command 主路径
- `services/local-minimal-node/src/node/access.rs`
  - `device` scope 现在只接受：
    - `device.telemetry`
    - `device.command`
  - `device.telemetry`：
    - 写入必须来自绑定设备 actor
    - 读取必须来自绑定设备 actor 或具备 `device.telemetry.read`
  - `device.command`：
    - 写入必须具备 `device.command.send`
    - 读取允许绑定设备 actor 消费目标命令流
  - 所有 `device` scope stream 都要求设备已经绑定到当前 principal，而不是继续接受任意 tenant 内部猜测 stream id 的访问。

### 3. 让 `device.telemetry` 真实复用 `DeviceSubject::sender(...)`
- `services/streaming-service/src/lib.rs`
  - `device.telemetry` 在 frame append 时不再直接使用 `auth.actor_id` 生成 sender。
  - 现在通过 `DeviceSubject::sender(None)` 物化：
    - `sender.id = device_id`
    - `sender.kind = "device"`
    - `sender.device_id = device_id`
    - `sender.session_id = auth.session_id`
- 这把设备主路径真正接回了 `CP08-1` 刚冻结的统一主体模型。
