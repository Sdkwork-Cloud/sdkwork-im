## 本轮实际完成

- `crates/sdkwork-im-runtime-link` 现在直接依赖 `sdkwork-im-ccp-registry`，runtime hello 协商开始消`EffectiveProtocolSnapshot`
- 新增 `LinkHelloPolicy`，把以下协商基线统一收口到一个只读对象：
  - `protocol`
  - `allowed_bindings`
  - `capabilities`
- `LinkSession::new(...)` 不再走本地硬编码 capability helper，而是通过 `default_hello_policy()` `CcpRegistry::control_plane_v1().governance_snapshot().effective_snapshot` 构造默认策略
- 新增 `LinkSession::new_with_effective_snapshot(...)`，用于测试与后续 runtime 注入场景，显式把 effective snapshot 注入 link session
- `negotiate_hello(...)` 已改为使snapshot 驱动协商
  - snapshot 中的协议主版本做校验
  - snapshot 中的 allowed bindings binding 校验
  - snapshot 中的 enabled capabilities 计算 hello_ack capability 交集
  - ack 协议版本返回 snapshot 定义的权威版本，而不再回落到本地默认
- 清理了旧的默认测试假设：
  - 默认 hello baseline 已按 control-plane snapshot 断言
  - 默认拒绝binding 已改为当kill switch 实际禁用`ccp/mqtt/1`
