真实入口已消费统一 CCP 运行时：

- `session-gateway` WebSocket 主入口
- `local-minimal-node` 的 realtime WebSocket 透传入口
- `tools/chat-cli` 的内部 realtime transport/runtime

已兑现的入口能力：

- 子协议 `craw-chat.ccp.ws.v1`
- `LegacyJson / CcpJson` 双模式兼容
- 服务端 `hello -> hello_ack -> auth_bind -> auth_ok`
- 业务帧统一通过 CCP envelope 承载

### 3.3 权威字段边界

- `im-auth-context` 已提供 `ccp_authority()`

- `realtime.connected` 中的 `actor / sender` 已由服务端生成
- 客户端不再在业务协议中提交权威 `tenantId / sender`
