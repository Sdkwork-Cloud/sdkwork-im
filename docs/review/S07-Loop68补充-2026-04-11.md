## 2. 本轮交付
- `PendingSharedChannelSyncRequest` 当前新增读取态 `lease_status(now)` 派生逻辑

- pending inventory item 当前显式返回 `leaseStatus`
- dead-letter inventory response model 当前也通过共享 response helper 暴露 `leaseStatus`

- 本轮没有改变 claim / release / takeover 的执行语义；补的是 operator 可见性，不是 policy engine

## 3. 运行时边界
- 本轮补的是 inventory/operator surface 派生状态，不是 stale policy 执行
- 当前仍没有：
  - `takeoverEligible` derived field
  - configurable SLA / lease threshold contract
  - automatic stale detection / timeout reclaim
  - stale-aware `repair-shared-channel-sync`
  - release-ready exactly-once semantics
