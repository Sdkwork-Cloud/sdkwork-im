# S05 Loop34补充 - 2026-04-10

## 1. 本轮主任务
- 修复 runtime-dir social durable truth 的 `snapshot ahead of journal` 幻影真相，收敛 crash consistency 边界。

## 2. 已兑现事实
- `SocialControlRuntime::load_state_with_journal_replay(...)` 已改为：`journal exists -> replay from default -> rewrite snapshot`；`journal missing -> fallback snapshot`。
- `SocialControlRuntime::persist_state_transition(...)` 已改为：`append journal -> save snapshot`。
- 新增并转绿：
  - `test_control_plane_social_file_runtime_discards_friend_request_snapshot_ahead_of_journal`
  - `test_control_plane_social_file_runtime_discards_direct_chat_snapshot_ahead_of_journal`
- 既有 replay 回归继续通过：
  - `test_control_plane_social_file_runtime_replays_friend_request_when_snapshot_is_missing`
  - `test_control_plane_social_file_runtime_replays_direct_chat_pair_guard_when_snapshot_is_missing`

## 3. 仍未关闭
- 仍无真正 `atomic tx boundary`
- 仍缺 `snapshot save fail` 的 failpoint / retry / repair 证据
- 仍缺 operator 级 replay / roll-forward / repair 闭环
- `S05 = not_closed / local_closure`

## 4. 当前结论
- Loop34 已关闭 `snapshot ahead of journal` 的 phantom truth 缺口；`S05` 主缺口收敛为两文件事务边界与 repair 闭环，而不再是启动 authority 错位。
