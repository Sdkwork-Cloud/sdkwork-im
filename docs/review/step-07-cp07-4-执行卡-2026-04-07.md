## 本轮实际完成

### 1. control-plane 写操作接入治理 side effect

- `services/control-plane-api` 在 `AppState` 中新增可选 `GovernanceLoop` sinks。
- 新增 `build_app_with_cluster_and_governance_sinks(...)`，允许 control-plane 显式接入 `OpsRuntime` 与 `AuditRuntime`。
- 在不改变原有 `control.write` 权限边界的前提下，现有写入入口开始同步触发治理 side effect：
  - `drain_node(...)`
  - `activate_node(...)`
  - `migrate_node_routes(...)`

### 2. ops / audit 获得最小治理闭环

- `mirror_node_into_ops_runtime(...)` 会把 realtime cluster 中的 node lifecycle 与 route ownership 镜像到 `OpsRuntime`。
- `record_control_plane_audit(...)` 会把 control-plane 写操作记录成 audit anchor，并保留 admin `AuthContext` 作为审计 actor。
- 本轮已接入的审计动作：
  - `control.node_draining_marked`
  - `control.node_activated`
  - `control.node_routes_migrated`

### 3. ops sink 加入节点归属保护

- `services/ops-service` 新增 `node_id()`，用于 control-plane 精确判断当前 sink 是否属于目标节点，避免把别的节点状态误写入当前 `OpsRuntime`。
