## 本轮已兑现能力
- `09`
  - `Wave C / Step 08 / CP08-3` 已在真实代码中打通 device telemetry / command 最小主路径
- `134`
  - 设备 telemetry 已复用统一 `StreamFrame + DeviceSubject::sender(...)`
  - 设备 command 已复用统一 stream 主链路，而不是独立命令面
- `139`
  - `device.command.send`
  - `device.telemetry.read`
  - 两个能力名已变成真实 capability gate
- `143`
  - `device` scope stream 不再允许绕过 authorizing gateway 直接命中 standalone streaming-service
- `145`
  - `device.telemetry` 已开始稳定对应 `cc.device.telemetry.v1`
  - `device.command` 已开始稳定对应 `cc.device.command.v1`
  - `sender` 继续由服务端基于权威上下文与设备主体生成，而不是信任客户端自报完整设备 sender 快照

## 本轮未兑现能力
- MQTT binding 的设备真实接入
- device twin / shadow 状态收口
- command ACK / timeout / retry 等更复杂设备协议语义
- `Step 09` 范围内的长期 durability / projection / observability 能力

## 是否偏离架构
- 无偏离。
- 本轮实现进一步验证了 `Step 08` 的核心判断：
  - 设备能力可以在统一 stream / sender / capability 体系内落地
  - 不需要为 IoT 旁路再造一套独立实时协议
