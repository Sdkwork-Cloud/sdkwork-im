## 3. TDD / Debugging Notes
- Red started with `repair_json["reclaimed"] = null`.
- During green, the implementation first hit a compile-time gap because `SocialSharedChannelSyncLeaseStatus` could not be compared directly.
- The targeted test then exposed a setup issue: mutating persisted stale lease data required rebuilding the repair control surface from `runtime_dir` so repair could read the persisted stale state.
- The final green contract is explicit:
  - pre-repair inventory shows `leaseStatus = stale`
  - repair returns `status = repaired`
  - repair returns `reclaimed = 1`
  - pending backlog is emptied after dispatch
