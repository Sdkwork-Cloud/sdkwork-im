# 152CJ Loop67 补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“manual takeover 已存在，但不会区分 active lease 与 expired lease”推进到“targeted takeover 只允许接管 expired foreign claim”的最小 stale-lease guard seam

## 2. As-Built 事实
- `control-plane-api` 当前在 `PendingSharedChannelSyncRequest` 上新增 `blocks_foreign_takeover(...)` 判定
- `SocialControlRuntime::takeover_pending_shared_channel_sync_targeted(...)` 当前会在 mutate state 之前先筛选 selected pending item
- 当 selected request 被其他 operator claim 且 `leaseExpiresAt > now` 时，当前会返回 `409 shared_channel_sync_owner_conflict`
- 当 selected request 的 `leaseExpiresAt <= now` 时，当前仍可沿用既有 targeted takeover 路径刷新 `ownerActorId / ownerActorKind / claimedAt / leaseExpiresAt`
- `leaseExpiresAt` 缺失的 legacy pending claim 当前不会阻断 takeover；本轮只补最小 stale guard，不扩成 migration / rewrite pass
- 协作测试当前通过 runtime-dir 手工回写 `social-state.json`，显式证明“active lease 拒绝 + expired lease takeover 成功”的运行时合同

## 3. 明确边界
- 本轮没有：
  - `leaseStatus / takeoverEligible` inventory derived field
  - configurable lease window / SLA
  - timeout reclaim / automatic stale detection
  - stale-aware bulk `repair-shared-channel-sync`
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 内部的 targeted stale-lease guard，而不是完整的 cross-service lease policy engine
