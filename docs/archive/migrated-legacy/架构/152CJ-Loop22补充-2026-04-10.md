# 152CJ Loop-22 补充

## 本轮新增事实
- `automation-service` 已补齐最小 automation governance surface：
  - `GET /im/v3/api/automation/governance`
  - `capabilityProfileId = stable-agent`
  - `enabledCapabilities = agent.response / agent.tool_call`
  - `guardrailPolicyId = automation-tool-call-guardrail-v1`
  - `restrictedToolPrefixes = ops. / admin.`
  - `operatorOverridePermission = automation.operator_override`
- 受限 tool call 默认被 guardrail 拦截；override 由调用方权限显式激活，而不是旁路跳过主链。
- `sdkwork-im-server` assembled 面已同步暴露 governance surface，并把 deny/override 决策写入 audit anchor。

## 当前准确口径
- `S11` 不再只是 `local_closure`。
- `S11` 当前可以诚实写为 `step_closure = agent lifecycle HTTP mainline + journal audit + guardrail deny + operator override + public governance surface + assembled audit anchors`。
- 本轮未宣称 control-plane 可写、可版本化、可回滚的 automation governance durable truth；若后续需要，应作为新的增量设计，不回滚当前 `S11` 闭环结论。
