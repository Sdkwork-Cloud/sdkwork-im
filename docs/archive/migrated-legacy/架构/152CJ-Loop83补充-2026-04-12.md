# 152CJ Loop83 Addendum - 2026-04-12

## 1. As-Built Update
- `PendingSharedChannelSyncRequest` now distinguishes between:
  - active same-owner repeat claim, which preserves the current lease
  - stale same-owner re-claim, which refreshes `claimedAt / leaseExpiresAt`
- A successful targeted claim by the current owner now implies an active lease window, not just preserved owner identity.
- No `deployment_profile`, script entry, or operator route changed in this loop.

## 2. Architectural Meaning
- Before Loop83, the targeted claim surface could return success while leaving a same-owner request in `leaseStatus = stale`.
- After Loop83, claim success for the current owner is lease-consistent:
  - stale same-owner claim renews the lease
  - active same-owner claim remains idempotent
- This is still a path-local normalization seam, not a background scheduler or timeout sweeper.

## 3. Deferred Architecture Work
- automatic stale detection / timeout reclaim / scheduler
- release-ready exactly-once delivery semantics for shared-channel sync
- richer stale-age / SLA policy beyond the current path-local renew / reclaim seams
