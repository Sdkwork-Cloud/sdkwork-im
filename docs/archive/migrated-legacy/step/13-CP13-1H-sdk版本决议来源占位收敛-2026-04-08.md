# 13-CP13-1H - SDK 版本决议来源占位收敛

## 本轮目的

SDK release catalog 里“未来版本冻结决议会从哪里落证据”这件事先冻结成 machine-readable 字段，哪怕当前真实决议来源还不存在，也要明确保留一个不会漂移的占位槽位

## 发现的问

- `sdk-release-catalog.json` 已显式保留：
  - `plannedVersion = null`
  - `versionStatus = version_unassigned_pending_freeze`
- 但还没有字段表达“未来版本冻结决议来源路径
- 结果是：当前虽然知道版本尚未冻结，但后续真实 freeze evidence 应写到哪里，仍没有结构化 contract

## 本轮决策

- 继续使用 `tools/chat-cli/tests/chat_cli_contract_test.rs` contract gate
- 新增 `test_continuous_optimization_sdk_release_catalog_freezes_version_decision_source_contract`
- 最小要求：
  - schema 必须定义 `versionDecisionSourcePath`
  - 当前 catalog 中每SDK artifact 都必须保`versionDecisionSourcePath = null`
  - `sdks/README.md`、`artifacts/releases/README.md`、`bundle-manifest.md` 必须同步说明该占位字

## 实施结果

- 更新
  - `artifacts/releases/schemas/sdk-release-catalog.schema.json`
  - `artifacts/releases/wave-d-2026-04-08/sdk-release-catalog.json`
  - `sdks/README.md`
  - `artifacts/releases/README.md`
  - `artifacts/releases/wave-d-2026-04-08/bundle-manifest.md`
- 现在 bundle release catalog 已同时冻结三组信息：
  - 当前版本值还未分配：`plannedVersion = null`
  - 当前版本状态仍freeze：`versionStatus = version_unassigned_pending_freeze`
  - 当前版本 freeze 决议来源路径尚未分配：`versionDecisionSourcePath = null`
