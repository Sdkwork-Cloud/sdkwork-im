# 13-CP13-1G - SDK 容器 README 发布边界收敛

## 本轮目的

把 `sdk-release-catalog.json` 的发布边界，从 bundle、workspace 根入口和四个叶子 README 继续补到 app/admin 两个容器 README，消除 SDK 导航链中间层对“尚未生成、尚未发布”状态的表达断层。

## 发现的问题

- 四个叶子 README 已公开：
  - `artifacts/releases/wave-d-2026-04-08/sdk-release-catalog.json`
  - `generationStatus = template_only_pending_generation`
  - `releaseStatus = not_published`
- 但 `sdks/sdkwork-craw-chat-sdk/README.md` 与 `sdks/sdkwork-craw-chat-sdk-admin/README.md` 只公开了版本占位，没有继续公开 release catalog 发布边界
- 结果是：SDK 导航链在容器层仍然看不到 bundle 内的唯一发布真源

## 本轮决策

- 继续使用 `tools/chat-cli/tests/chat_cli_contract_test.rs` 作为 contract gate
- 新增 `test_continuous_optimization_sdk_container_readmes_freeze_release_catalog_boundary`
- 最小要求：
  - 两个容器 README 都必须公开：
    - `artifacts/releases/wave-d-2026-04-08/sdk-release-catalog.json`
    - `generationStatus = template_only_pending_generation`
    - `releaseStatus = not_published`

## 实施结果

- 更新：
  - `sdks/sdkwork-craw-chat-sdk/README.md`
  - `sdks/sdkwork-craw-chat-sdk-admin/README.md`
- 两个容器 README 现在都显式说明：
  - bundle 内 `sdk-release-catalog.json` 是当前 SDK 发布真源
  - 真实生成状态仍是 `template_only_pending_generation`
  - 真实发布状态仍是 `not_published`

## 验证

- 红灯：
  - `cargo test -p craw-chat-cli --offline --test chat_cli_contract_test test_continuous_optimization_sdk_container_readmes_freeze_release_catalog_boundary -- --nocapture`
  - 失败点：`app SDK container README must contain release catalog boundary text sdk-release-catalog.json`
- 绿灯：
  - `cargo fmt --all --check`
  - `cargo test -p craw-chat-cli --offline --test chat_cli_contract_test test_continuous_optimization_sdk_container_readmes_freeze_release_catalog_boundary -- --nocapture`
  - `cargo test -p craw-chat-cli --offline --test chat_cli_contract_test -- --nocapture`

## 下一轮建议

- SDK release 链下一个更小的真实缺口已转向：
  - 版本冻结决议来源的结构化表达
  - 真实生成输入与真实发包归档
