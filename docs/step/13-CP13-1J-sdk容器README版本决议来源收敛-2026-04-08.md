# 13-CP13-1J - SDK 容器 README 版本决议来源收敛

## 本轮目的

把 `versionDecisionSourcePath = null` 从 bundle、总入口和叶子 README 继续补到 app/admin 两个容器 README，彻底打通 SDK release 导航链里关于“版本决议来源尚未分配”的占位表达。

## 发现的问题

- `sdks/README.md` 与四个叶子 README 已公开：
  - `versionDecisionSourcePath = null`
- 但 `sdks/sdkwork-craw-chat-sdk/README.md` 与 `sdks/sdkwork-craw-chat-sdk-admin/README.md` 两个容器入口还没有同步这条占位语义
- 结果是：导航链在中间层再次出现版本决议来源字段断层

## 本轮决策

- 继续使用 `tools/chat-cli/tests/chat_cli_contract_test.rs` 做 contract gate
- 新增 `test_continuous_optimization_sdk_container_readmes_freeze_version_decision_source_boundary`
- 最小要求：
  - 两个容器 README 都必须公开：
    - `versionDecisionSourcePath`
    - `null`

## 实施结果

- 更新：
  - `sdks/sdkwork-craw-chat-sdk/README.md`
  - `sdks/sdkwork-craw-chat-sdk-admin/README.md`
- 现在容器 README 也显式对齐：
  - `plannedVersion = null`
  - `versionStatus = version_unassigned_pending_freeze`
  - `versionDecisionSourcePath = null`
