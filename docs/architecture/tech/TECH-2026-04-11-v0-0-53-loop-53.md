# v0.0.53 - Loop 53 - 2026-04-11

## 本轮目标
- 把 `control-plane ready-pair auto-trigger seam` 接到 `local-minimal-node` 的真实 `ConversationRuntime` consumer，形成 embedded 装配面的最小闭环

## 本轮实现
- `control-plane-api`
  - 抽出 embedded control surface builder，供组合装配面直接 merge `/backend/v3/api/control/*`
  - 新增 `runtime_dir + governance_sinks + shared_channel_sync_trigger` 组合 builder
- `local-minimal-node`
  - 生产依赖 `control-plane-api`
  - 默认/公开装配面 merge embedded control-plane surface
  - 新增同进程 `SharedChannelLinkedMemberSyncTrigger`，直接调用 `ConversationRuntime::sync_shared_channel_linked_member(...)`
  - `OpsRuntime` service inventory 现显式包含 `control-plane-api`
- `projection-service`
  - `timeline_from_auth_context(...)` 接受 `can_read_shared_history()` 的 linked member 读取 shared history
- 测试
  - 新增 `services/local-minimal-node/tests/control_plane_social_sync_e2e_test.rs`
  - 用 runtime-dir seed shared-history conversation，再通过 merged control-plane 写入 `external_connection / shared_channel_policy / external_member_link`，验证 linked actor 能在 local-minimal timeline 上读到历史

## 风险与未闭环
- standalone `control-plane-api::build_app*` / `services/control-plane-api/src/main.rs` 仍未装配真实 shared-channel sync consumer
- 当前没有 outbox / retry / remote republish，仍不是跨服务 exactly-once 闭环
- 本轮只放宽了 projection timeline history read；其他 linked-member read surface 是否同步放宽，当前未宣称

## 结论
- `S07` 仍是 `not_closed / local_closure`
