# S07 Loop47补充 - 2026-04-11

## 1. 本轮唯一主缺口
- Loop47 固定只做 `shared-external history runtime truth`
- `retention enforcement`、`thread subscription / notification`、`shared fanout / rebuild` 均保持 deferred，不与本轮并行混写

## 2. 本轮交付
- `ConversationPolicy::normalize` 正式接受 `history_visibility = shared`
- `conversation-runtime` 的 HTTP `add-member` surface 新增 `attributes` 透传
- 带完整 `sharedChannelPolicyId + externalConnectionId + externalMemberId` 锚点的新成员会落为 `MembershipState::Linked`
- `Linked` 成员现在可以读取 shared-history，但仍不是 generic active member，不能直接发消息

## 3. 运行时边界
- 本轮 shared truth 是“显式锚点驱动”的最小 runtime 事实，不是 control-plane 自动同步
- local joined 成员仍保持正常读写；external-linked 成员只获得历史读取能力
- 不能把 `shared_channel_policy` durable truth 误写成“runtime 已自动消费”
