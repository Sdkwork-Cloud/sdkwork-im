## 异常点
- 当前 `drain-rebalance` 和 `failover` 结果都基于单 route / 单 device 场景，不能外推为大规模多租户环境结论。
- 当前 `restore-recovery` 仍是本地 runtime-dir snapshot 路径，不代表 object storage / archive / cross-region 恢复已经完成。
- 当前 `upgrade-rollback` 只验证最小可信的 kill switch rollback 路径，不代表多 cell / 多 region 的真实编排已经完成。

## 当前结论
- `CP11-3` 已满足最小本地演练要求：
  - drain / rebalance
