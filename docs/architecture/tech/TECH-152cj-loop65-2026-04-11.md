# 152CJ Loop65 补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“pending/dead-letter inventory 已显式暴露 `claimedAt`，但 foreign-owned claim 仍无法显式接管”推进到“control-plane 提供最小 manual targeted takeover contract”

## 2. As-Built 事实
- `control-plane-api` 当前新增 `POST /backend/v3/api/control/social/runtime/takeover-pending-shared-channel-sync-targeted`
- `SocialControlRuntime::takeover_pending_shared_channel_sync_targeted(...)` 当前只会转移 selected 且 `foreign-owned` 的 pending request
- takeover 成功时，owner 元数据会切换为当前 operator，并刷新 durable `claimedAt`
- unclaimed / self-owned / missing selected request 当前保持 `noop`，不引入额外错误语义
- takeover 后，旧 owner 再走 `republish-pending-shared-channel-sync-targeted` 会继续命中 `409 shared_channel_sync_owner_conflict`；新 owner 可复用既有 targeted republish 路径

## 3. 明确边界
- 本轮没有：
  - `leaseExpiresAt / SLA`
  - stale-age threshold / automatic stale detection
  - background retry scheduler
  - ownership-aware `repair-shared-channel-sync`
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 单体内的 manual takeover seam，而不是 release-ready stale-claim orchestration
