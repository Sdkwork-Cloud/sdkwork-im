# 152CJ Loop52补充 - 2026-04-11

- `control-plane-api` 现在为 `external_member_link` durable truth 显式持久化 `localActorKind`
  - 该字段进入 control-plane request、durable snapshot、social commit payload 与 replay truth
  - 这让 `shared_channel_policy + external_member_link` ready pair 首次具备完整 `conversation-runtime` sync payload 所需的 principal-kind bridge
- `control-plane-api` 新增 `SharedChannelLinkedMemberSyncRequest` 与 `SharedChannelLinkedMemberSyncTrigger`
  - 该 seam 由装配层显式注入，避免 `control-plane-api` 直接依赖 `conversation-runtime`
  - trigger payload 固定覆盖 `conversationId / sharedChannelPolicyId / externalConnectionId / localActorId / localActorKind / externalMemberId`
- control-plane social 写路径的 As-Built 边界更新为：
  - `bind_external_member_link(...)`：当同连接上已存在 active shared-channel policy 且带 `conversationId` 时，会自动 dispatch sync trigger
  - `apply_shared_channel_policy(...)`：当同连接上已存在 active external-member-links 且带 `localActorKind` 时，会为每条 ready link dispatch sync trigger
  - success dispatch 会沉淀 `control.shared_channel_linked_member_sync_triggered` audit anchor
- 当前仍然不能把系统解释为“control-plane 默认即自动 materialize runtime linked member”
  - 默认 `build_app*` builder 仍未装配真实 trigger implementation
  - 当前没有 outbox / retry / cross-service republish，因此这仍是 injected seam，不是 release-ready distributed closure
  - legacy 缺失 `localActorKind` 的 `external_member_link` truth 也不会自动补齐
- As-Built 结论现在更新为：
  - 已闭合到 `control-plane trigger seam`：`shared_channel_policy + external_member_link -> ready-pair resolution -> sync request dispatch`
  - 仍待闭合到 `real runtime consumer wiring / exactly-once orchestration`
