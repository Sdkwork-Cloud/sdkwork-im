# S07 Loop66补充 - 2026-04-11

## 1. 本轮唯一主缺口
- Loop66 固定只做 `pending leaseExpiresAt visibility`
- `stale-age threshold / SLA / scheduler / https:// public runtime target` 继续 deferred，不与本轮并行混做

## 2. 本轮交付
- `PendingSharedChannelSyncRequest` 当前新增 durable `leaseExpiresAt`
- `GET /backend/v3/api/control/social/runtime/pending-shared-channel-sync` inventory item 当前显式返回 `leaseExpiresAt`
- `GET /backend/v3/api/control/social/runtime/dead-letter-shared-channel-sync` 复用同一 inventory item 结构，也会返回 `leaseExpiresAt`
- owner claim 当前会按固定 `15m` lease window 写入 `leaseExpiresAt`
- targeted takeover 当前会和 `claimedAt` 一起刷新 `leaseExpiresAt`
- owner-only targeted release 当前会清空 `ownerActorId / ownerActorKind / claimedAt / leaseExpiresAt`
- 同 owner 重复 claim 当前保留原 `claimedAt / leaseExpiresAt`，不扩展成 lease refresh 语义

## 3. 运行时边界
- 本轮补的是 lease metadata visibility，不是 stale-age / SLA 执行
- 当前仍没有：
  - stale-age threshold / timeout policy
  - automatic stale detection / timeout reclaim
  - ownership-aware `repair-shared-channel-sync`
  - release-ready exactly-once semantics
