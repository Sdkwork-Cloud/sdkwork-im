# v0.0.67 - Loop 67 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“leaseExpiresAt 已可见，但 manual takeover 不区分 active / expired lease”推进到“operator 只能 takeover expired foreign claim”的最小 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - `PendingSharedChannelSyncRequest` 新增 active-lease takeover guard
  - `takeover_pending_shared_channel_sync_targeted(...)` 当前会在 mutate state 前阻断 `leaseExpiresAt > now` 的 foreign claim
  - active foreign lease 当前返回 `409 shared_channel_sync_owner_conflict`
  - expired foreign lease 仍可沿用既有 targeted takeover 路径完成接管
  - `leaseExpiresAt` 缺失的 legacy pending claim 当前保持可 takeover 的最小兼容边界
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 收紧 Loop65 takeover e2e，先验证 active lease 被 `409` 阻断
  - 同一条 e2e 通过 runtime-dir 回写过期 `leaseExpiresAt`，再验证 expired lease 可被 takeover
  - 新增测试 helper，把 `social-state.json` 回写为可控的过期 lease 状态

## 风险与未闭环
- 当前 stale 判定仍绑定固定 `15m` lease window，尚未升级为 operator-visible SLA / threshold contract
- 当前 inventory surface 仍未直接给出 `stale / active / takeoverEligible` 派生状态
- 当前没有 automatic stale detection / timeout reclaim / scheduler
- 当前 `repair-shared-channel-sync` 仍是粗粒度全量 route，尚未并入 stale-aware policy
- 当前没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth

## 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop67 已把 `shared-channel sync` 补齐到 `durable pending backlog + operator repair + pending inventory + pending targeted republish + pending targeted claim + republish ownership guard + pending targeted release + pending claimedAt visibility + pending targeted takeover + pending leaseExpiresAt visibility + pending stale-lease takeover guard + next healthy ready-pair write auto-retry + repeated-failure dead-letter + dead-letter inventory + dead-letter targeted/all requeue + failure-budget rearm`
- 下一主缺口收敛为 `operator stale-status visibility / explicit stale-evaluated metadata`
