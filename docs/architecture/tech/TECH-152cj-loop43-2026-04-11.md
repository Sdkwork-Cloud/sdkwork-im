# 152CJ Loop43补充 - 2026-04-11

- `S05` 的 `repair-marker` 边界现在不只存在于运行时实现，也进入了显式 operator contract：
  - `SocialRuntimeRepairResponse` 新增 `transactionMarkerCleared`
  - HTTP `repair-derived-snapshot` 与 standalone `repair-social-runtime-dir` CLI 都会直接报告 pending marker 是否在本次修复中被清理
