# 152CJ Loop63 补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“pending request 已有 owner claim，但 claim 无显式 release lifecycle”推进到“owner-only targeted release + reclaim”的最小 claim lifecycle 语义

## 2. As-Built 事实
- `control-plane-api` 当前新增 `POST /backend/v3/api/control/social/runtime/release-pending-shared-channel-sync-targeted`
- `SocialControlRuntime::release_pending_shared_channel_sync_targeted(...)` 当前会：
  - 在 release 前校验 selected pending item 是否归当前 operator 所有
  - 对 foreign-owned request 返回 `shared_channel_sync_owner_conflict`
  - 对 owner-owned request 清空 `ownerActorId / ownerActorKind`
  - 保持 pending item 本身与既有 failure budget / dead-letter 语义不变
- released request 当前会回到 unowned pool，因此其他 operator 可以再次 claim，并复用既有 targeted republish
- 现有 `claim-pending-shared-channel-sync-targeted`、`republish-pending-shared-channel-sync-targeted` 与 `release-pending-shared-channel-sync-targeted` 当前共同组成最小 claim lifecycle operator surface

## 3. 明确边界
- 本轮没有：
  - `claimedAt / leaseExpiresAt / SLA` 元数据
  - foreign operator 的 force release / takeover
  - ownership-aware `repair-shared-channel-sync`
  - automatic retry scheduler
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 单体内的 manual claim lifecycle boundary，而不是 release-ready stale-claim / lease orchestration
