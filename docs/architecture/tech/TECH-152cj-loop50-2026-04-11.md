# 152CJ Loop50补充 - 2026-04-11

- `projection-service` 新增 `message_posted_notification_principal_ids_from_auth_context(...)`
  - message-posted recipient resolution 继续由 projection owner seam 负责，而不是把 recipient set 交回 service edge 拼装
  - 该 seam 当前返回 `joined active members + shared-history-visible linked members`
  - linked 成员是否进入 fanout，取决于已投影的 `ConversationMember.state = linked` 与 `sharedChannelPolicyId / externalConnectionId / externalMemberId` 三个锚点是否完整
- `notification-service` 现已把 `request_message_posted_notifications(...)` 切到该 seam
  - shared linked principal 会收到 message-posted notification
  - actor 仍在 notification fanout 阶段被排除
- replay / rebuild 边界明确为：
  - projection 继续完整持久化 `ConversationMember`
  - 事件重放后，shared linked recipients 仍会进入同一 notification target 集合
  - 本轮不需要新增 rebuild 专用存储模型
- As-Built 边界继续保持：
  - 已闭合：`reaction / pin`、`direct_chat -> conversation`、`thread minimal runtime truth`、`thread root-author subscription / notification runtime truth`、`invited history visibility`、`shared-external history`、`shared fanout / rebuild`、`retention enforcement runtime truth`
  - 仍待闭合：`shared_channel_policy auto-projection / durable sync`
  - 更强 `im_thread_subscription` durable model / per-user unread-notification level 继续 deferred
