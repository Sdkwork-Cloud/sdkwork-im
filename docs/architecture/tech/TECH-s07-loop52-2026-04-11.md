# S07 Loop52补充 - 2026-04-11

## 1. 本轮唯一主缺口
- Loop52 固定只做 `control-plane shared durable truth -> runtime sync trigger seam`
- `real assembly wiring / outbox / retry`、更强 `im_thread_subscription` durable model、per-user unread / notification level 均保持 deferred，不与本轮并行混写

## 2. 本轮交付
- `external_member_link` durable truth 新增 `localActorKind` bridge
- `control-plane-api` 新增 `SharedChannelLinkedMemberSyncRequest` 与 `SharedChannelLinkedMemberSyncTrigger`
- `bind_external_member_link(...)` 和 `apply_shared_channel_policy(...)` 现在都会在 ready pair 成立时自动解析并 dispatch shared-channel linked-member sync request
- auto-trigger payload 至少覆盖 `conversationId / sharedChannelPolicyId / externalConnectionId / localActorId / localActorKind / externalMemberId`
- trigger 成功 dispatch 后会写入 `control.shared_channel_linked_member_sync_triggered` audit anchor

## 3. 运行时边界
- 本轮没有让 `control-plane-api` 直接依赖 `conversation-runtime`
- 默认 `build_app*` builder 仍不注入 trigger；只有显式装配 `SharedChannelLinkedMemberSyncTrigger` 的场景才会真正把请求送到 runtime consumer
- 当前没有 outbox / retry / remote republish；dispatch 仍是进程内注入 seam，不是跨服务闭环
- legacy `external_member_link` 若缺失 `localActorKind`，当前不会参与新的 auto-trigger
