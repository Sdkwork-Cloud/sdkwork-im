# 152CJ Loop64 补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“operator 只能看到 owner，无法观察 claim 时间”推进到“pending/dead-letter inventory 显式暴露 `claimedAt`”的最小 stale-claim 可观测性语义

## 2. As-Built 事实
- `control-plane-api` 当前为 `PendingSharedChannelSyncRequest` 新增 durable `claimedAt`
- `SocialControlRuntime::claim_pending_shared_channel_sync_targeted(...)` 当前会在首次 owner claim 时写入 UTC RFC3339 毫秒精度 `claimedAt`
- 同 owner 重复 claim 当前保留原 `claimedAt`，不把“重复 claim”解释成 lease refresh
- `SocialControlRuntime::release_pending_shared_channel_sync_targeted(...)` 当前会在 owner release 时一并清空 `ownerActorId / ownerActorKind / claimedAt`
- `GET /backend/v3/api/control/social/runtime/pending-shared-channel-sync` 与 `GET /backend/v3/api/control/social/runtime/dead-letter-shared-channel-sync` 当前都通过同一 inventory item 结构显式返回 `claimedAt`

## 3. 明确边界
- 本轮没有：
  - `leaseExpiresAt / SLA`
  - foreign operator force release / takeover
  - ownership-aware `repair-shared-channel-sync`
  - automatic retry scheduler
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 单体内的 claim-time observability seam，而不是 release-ready stale-claim orchestration
