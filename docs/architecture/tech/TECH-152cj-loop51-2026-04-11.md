# 152CJ Loop51补充 - 2026-04-11

- `conversation-runtime` 新增 `SyncSharedChannelLinkedMemberCommand`
  - durable truth 的最小 runtime 消费口径被收敛为一个 system-owned sync seam，而不是继续复用 generic `members/add`
  - sync seam 会把 `shared_channel_policy + external_member_link` 派生 payload materialize 为 `role = guest / state = linked` 的 runtime member
  - runtime replay 继续复用既有 `conversation.member_joined` envelope，不需要新增 recovery event type
- `conversation-runtime` 新增 `POST /im/v3/api/chat/conversations/shared-channel-links/sync`
  - HTTP authority capture 继续收敛在 runtime auth-context entrypoint
  - service edge 不再手工线程化 requester kind / member shape
- As-Built 边界现在更新为：
  - 已闭合：`reaction / pin`、`direct_chat -> conversation`、`thread minimal runtime truth`、`thread root-author subscription / notification runtime truth`、`invited history visibility`、`shared-external history`、`shared fanout / rebuild`、`shared_channel_policy durable sync seam`、`retention enforcement runtime truth`
  - 仍待闭合：`control-plane -> conversation-runtime automatic sync trigger / auto-projection`
  - 更强 `im_thread_subscription` durable model / per-user unread-notification level 继续 deferred
- 当前明确的架构边界：
  - `control-plane-api` 仍不直接依赖 `conversation-runtime`
  - `external_member_link` durable truth 仍不含 principal kind，因此 `localActorKind` 仍由 sync 调用方桥接
  - 现在不能把系统解释为“control-plane durable truth 一提交就自动 materialize linked member”；当前只是 runtime 侧已经具备可消费的 sync seam
