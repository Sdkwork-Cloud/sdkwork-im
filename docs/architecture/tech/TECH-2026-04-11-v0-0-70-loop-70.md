# v0.0.70 - Loop 70 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“legacy `untracked` claim 会被普通 takeover 路径隐式接管”推进到“legacy `untracked` claim 必须走显式 override”的最小 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - inventory item 新增 `legacyTakeoverRequired`
  - `takeoverEligible` 当前只表示普通 takeover route 可直接执行的资格
  - targeted takeover request 新增 `allowLegacyUntracked`
  - 普通 takeover 当前会对 foreign `untracked` claim 返回 `409 shared_channel_sync_legacy_takeover_override_required`
  - 显式 `allowLegacyUntracked = true` 当前仍允许 legacy compatibility takeover
  - takeover response 新增 `legacyOverrideUsed`
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 现有 takeover lifecycle 当前升级为：active foreign claim 默认拒绝、legacy `untracked` 默认拒绝、stale foreign claim 仍可普通 takeover
  - 新增 dedicated override test，锁定 `allowLegacyUntracked = true` 时的成功合同与 `legacyOverrideUsed = true`

## 风险与未闭环
- dead-letter inventory 当前复用同一 response helper 暴露 `legacyTakeoverRequired / takeoverEligible`，但没有单独的 parity 断言
- legacy override 当前还是 body-level explicit override，不是单独的治理审批 workflow
- 当前没有 automatic stale detection / timeout reclaim / scheduler
- 当前 `repair-shared-channel-sync` 仍不是 stale-aware policy
- 当前没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth
