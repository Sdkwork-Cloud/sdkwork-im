# 13-CP13-1D - SDK 版本冻结占位收敛

## 本轮目的

`sdk-release-catalog` 从“有发布状态”继续推进到“显式表达版本尚未冻结”的 machine-readable contract，避免后bundle README 各自猜测 SDK 版本号

## 发现的问

- 当前 `sdk-release-catalog.json` 已冻结：
  - package
  - generationStatus
  - releaseStatus
- 但还没有正式表达
  - 当前是否已有计划版本
  - 当前版本是否已经冻结
- 结果是：发布目录知道“未生成、未发布”，但还不能明确表达“版本号仍未分配

## 本轮决策

- 继续使用 `tools/chat-cli/tests/chat_cli_contract_test.rs` 作为 contract gate
- 新增 `test_continuous_optimization_sdk_release_catalog_freezes_version_placeholder_contract`
- 当前最小版本占contract 固定为：
  - `plannedVersion = null`
  - `versionStatus = version_unassigned_pending_freeze`

## 实施结果

- 更新 `artifacts/releases/wave-d-2026-04-08/sdk-release-catalog.json`
- 更新 `artifacts/releases/schemas/sdk-release-catalog.schema.json`
- 更新
  - `sdks/README.md`
  - `artifacts/releases/README.md`
  - `artifacts/releases/wave-d-2026-04-08/bundle-manifest.md`
- 当前公开入口都已明确：
  - 版本尚未冻结
  - 不能从当bundle 推断真实 SDK 版本

## 验证

- 红灯
  - `cargo test -p sdkwork-im-cli --offline test_continuous_optimization_sdk_release_catalog_freezes_version_placeholder_contract -- --exact --nocapture`
  - 失败点：`SDK release catalog schema must define versionStatus`
- 绿灯
  - `cargo fmt --all --check`
  - `cargo test -p sdkwork-im-cli --offline test_continuous_optimization_sdk_release_catalog_freezes_version_placeholder_contract -- --exact --nocapture`
  - `cargo test -p sdkwork-im-cli --offline --test chat_cli_contract_test -- --nocapture`

## 下一轮建

- 若继续推进真实版本冻结链，可`plannedVersion = null` 推进为：
  - 真实 semver
  - freeze 时间
  - freeze 决议来源
- 在真实版本分配前，继续保`version_unassigned_pending_freeze`，不伪造版本号
