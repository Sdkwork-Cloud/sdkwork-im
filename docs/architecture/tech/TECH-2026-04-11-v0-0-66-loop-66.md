# v0.0.66 - Loop 66 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“claimedAt 已可见，但 lease deadline 仍不可见”推进到“operator 可按 inventory 显式审计 `leaseExpiresAt`”的最小 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - `PendingSharedChannelSyncRequest` 新增 durable `leaseExpiresAt`
  - pending/dead-letter inventory item 显式返回 `leaseExpiresAt`
  - owner claim 会按固定 `15m` lease window 写入 `leaseExpiresAt`
  - targeted takeover 会和 `claimedAt` 一起刷新 `leaseExpiresAt`
  - owner-only targeted release 清空 `ownerActorId / ownerActorKind / claimedAt / leaseExpiresAt`
  - 同 owner 重复 claim 保留原 `claimedAt / leaseExpiresAt`
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 收紧 Loop63 release lifecycle e2e，验证 `leaseExpiresAt = null -> 非空 -> null`
  - 收紧 Loop65 takeover e2e，验证 foreign takeover 会刷新 `leaseExpiresAt`

## 风险与未闭环
- 当前 `leaseExpiresAt` 只是 operator-visible metadata，还没有 stale-age threshold / timeout policy
- 当前没有 automatic stale detection / timeout reclaim / scheduler
- 当前 `repair-shared-channel-sync` 仍是粗粒度全量 route，尚未并入 stale-age policy language
- 当前没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth

## 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop66 已把 `shared-channel sync` 补齐到 `durable pending backlog + operator repair + pending inventory + pending targeted republish + pending targeted claim + republish ownership guard + pending targeted release + pending claimedAt visibility + pending targeted takeover + pending leaseExpiresAt visibility + next healthy ready-pair write auto-retry + repeated-failure dead-letter + dead-letter inventory + dead-letter targeted/all requeue + failure-budget rearm`
- 下一主缺口收敛为 `stale-age threshold / operator judgment / SLA semantics`
