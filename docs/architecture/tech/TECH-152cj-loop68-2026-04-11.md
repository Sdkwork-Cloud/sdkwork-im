# 152CJ Loop68补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“operator 能看到 `leaseExpiresAt`，但仍需自行推导 lease 状态”推进到“inventory/operator surface 直接返回 `leaseStatus`”的最小 visibility seam

## 2. As-Built 事实
- `control-plane-api` 当前新增 `SocialSharedChannelSyncLeaseStatus` 枚举：`unclaimed / active / stale / untracked`
- `PendingSharedChannelSyncRequest` 当前新增 `lease_status(&self, now)`；读取 inventory 时会依据 `ownerActorId / ownerActorKind / leaseExpiresAt` 派生 lease 状态
- `SocialSharedChannelSyncInventoryItemResponse` 当前新增 `leaseStatus`
- pending inventory 与 dead-letter inventory 当前共用 `social_shared_channel_sync_inventory_item_response(...)` response helper，因此都会返回同一套 `leaseStatus` 字段
- `leaseExpiresAt` 缺失的 legacy claimed item 当前不会被解释为 `active` 或 `stale`，而是显式返回 `untracked`
- takeover / claim / release 的状态机语义本轮未变；本轮只增加 operator-visible derived metadata

## 3. 明确边界
- 本轮没有：
  - `takeoverEligible` inventory derived field
  - configurable lease window / SLA
  - timeout reclaim / automatic stale detection
  - stale-aware bulk `repair-shared-channel-sync`
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 内部的 visibility seam，而不是完整的 cross-service lease policy engine
