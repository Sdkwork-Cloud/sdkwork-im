# 152CJ Loop37补充 - 2026-04-10

- `S05` runtime-dir social durable truth 当前再向前收敛一格：
  - 新增 `state/social-failpoints.json` 的一次性 `failNextSnapshotSave`
  - `snapshot save fail` 可以稳定复现为 `journal committed + snapshot unavailable`
  - 新增 `POST /backend/v3/api/control/social/runtime/repair-derived-snapshot`，operator 可显式回刷 derived snapshot
  - failpoint、same-event retry、operator repair、重启读回三者已形成一条可审计的 repair/roll-forward 证据链
- 当前 `S05` 的剩余主缺口不再是 failpoint/repair 证据，而是 `atomic tx boundary` 与 `standalone journal-only operator repair/replay tooling`。
