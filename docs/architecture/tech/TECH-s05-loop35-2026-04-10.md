# S05 Loop35补充 - 2026-04-10

## 1. 本轮主任务
- 修复 `journal append` 成功但 `snapshot save` 失败后，live state 仍停留旧内存而继续误接收写入的问题。

## 2. 已兑现事实
- `SocialControlRuntime::persist_state_transition(...)` 当前语义为：
  - `append social-commit-journal`
  - 若 `save social-state` 成功，则正常返回
  - 若 `save social-state` 失败，则 live state 仍推进到 `next_state`，同时返回 `social_state_unavailable`
- 新增并保持通过：
  - `test_control_plane_social_file_runtime_keeps_direct_chat_pair_guard_after_snapshot_save_failure`
- fresh 包级回归：`cargo test -p control-plane-api --offline --tests -- --nocapture` = `55 passed`

## 3. 仍未关闭
- 仍无真正 `atomic tx boundary`
- 仍缺客户端 `503` 后的 ack / idempotent retry contract
- 仍缺 `snapshot save fail` 的 failpoint / retry / repair 证据
- 仍缺 operator 级 replay / roll-forward / repair 闭环
- `S05 = not_closed / local_closure`

## 4. 当前结论
- Loop35 已关闭“snapshot save fail 之后同进程继续跑旧状态”的语义缺口；`S05` 主缺口进一步收敛为客户端确认语义、故障注入证据与 operator repair 闭环。
