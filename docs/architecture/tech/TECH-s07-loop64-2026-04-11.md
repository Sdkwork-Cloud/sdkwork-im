# S07 Loop64补充 - 2026-04-11

## 1. 本轮唯一主缺口
- Loop64 固定只做 `pending claimedAt visibility`
- `leaseExpiresAt / takeover / scheduler / https:// public runtime target` 继续 deferred，不与本轮并行混做

## 2. 本轮交付
- `PendingSharedChannelSyncRequest` 当前新增 durable `claimedAt`
- `GET /backend/v3/api/control/social/runtime/pending-shared-channel-sync` inventory item 当前显式返回 `claimedAt`
- `GET /backend/v3/api/control/social/runtime/dead-letter-shared-channel-sync` 复用同一 inventory item 结构，也会返回 `claimedAt`
- owner claim 当前会写入 UTC RFC3339 毫秒精度 `claimedAt`
- owner-only targeted release 当前会清空 `ownerActorId / ownerActorKind / claimedAt`
- 同 owner 重复 claim 当前保留原 `claimedAt`，不扩展成 lease refresh 语义
- Loop63 release lifecycle e2e 当前已升级为 claimedAt 回归：
  - claim 前 inventory `claimedAt = null`
  - claim 后 inventory `claimedAt != null`
  - release 后 inventory `claimedAt = null`

## 3. 运行时边界
- 本轮补的是 stale-claim 可观测性，不是 lease / SLA 执行
- 当前仍没有：
  - `leaseExpiresAt / SLA` metadata
  - stale-claim automatic reclamation
  - foreign operator force release / takeover contract
  - ownership-aware `repair-shared-channel-sync`
  - release-ready exactly-once semantics

## 5. 结论
- `S07` 仍只能标记为 `not_closed / local_closure`
- Loop64 已把 shared-channel sync 从“只有 owner，可见不到 claim 时间”推进到“inventory 可显式审计 claim 时间”
