# 152CJ Loop41补充 - 2026-04-11

- `S05` runtime-dir social durable truth 再向前收敛一格：
  - `repair-runtime-local.ps1` / `repair-runtime-local.sh` 现已把 generic runtime repair 与 standalone social journal repair 统一为同一个 operator 入口
  - operator 不再需要在 generic repair 之后手工再执行一次 `control-plane-api repair-social-runtime-dir`
  - social journal 缺失时 wrapper 不误报失败；social journal 存在且追加 repair 失败时，wrapper 显式失败，避免产生“generic repair 成功但 social snapshot 仍未修复”的假阳性
- 当前 wrapper JSON 合同保持保守：
  - `--json` 继续保留 generic runtime-dir repair 的 stdout 载荷
  - social repair 作为附加 gate 执行，不额外扩展新的 wrapper-level JSON envelope
- 因 social 写路径仍非真正 `atomic multi-file tx`，且 `S05` 仍缺完整 step closure 证明，`S05` 继续只能标记为 `not_closed / local_closure`。
