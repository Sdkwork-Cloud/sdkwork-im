# 152CJ Loop85 Addendum - 2026-04-12

## 1. As-Built Update
- Targeted pending-republish now normalizes same-owner stale lease metadata before both exit shapes:
  - dispatch-enabled republish
  - `trigger_unconfigured` early return
- A same-owner operator republish attempt is now lease-consistent regardless of whether a runtime trigger is currently wired.
- No `deployment_profile`, script entry, or operator route changed in this loop.

## 2. Architectural Meaning
- Before Loop85, targeted republish had a trigger-dependent state-machine divergence:
  - dispatch-enabled paths renewed stale same-owner leases
  - trigger-unconfigured paths returned early and left stale metadata untouched
- After Loop85, explicit operator republish is normalized before trigger availability is evaluated.
- This keeps owner metadata semantics tied to the operator action itself, not to the downstream dispatch wiring state.

## 3. Deferred Architecture Work
- automatic stale detection / timeout reclaim / scheduler
- release-ready exactly-once delivery semantics for shared-channel sync
- richer proactive stale-age / SLA policy beyond current operator-triggered and path-local normalization seams
