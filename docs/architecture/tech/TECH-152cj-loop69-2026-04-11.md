# 152CJ Loop69补充 - 2026-04-11

## 1. 本轮唯一架构增量
- 把 `shared-channel sync` 从“inventory 已返回 `leaseStatus`，但 operator 仍需自己判断当前 actor 能否接管”推进到“inventory/operator surface 直接返回 `takeoverEligible`”的最小 visibility seam

## 2. As-Built 事实
- `control-plane-api` 当前在 `SocialSharedChannelSyncInventoryItemResponse` 上新增 `takeoverEligible`
- `PendingSharedChannelSyncRequest` 当前新增 `takeover_eligible_for(actor_id, actor_kind, now)`
- pending inventory 与 dead-letter inventory 当前都会基于“当前 actor + `control.write` 权限 + 当前 owner/lease 状态”返回 `takeoverEligible`
- `takeoverEligible = false` 当前覆盖：
  - unclaimed request
  - self-owned request
  - active foreign claim
  - 无 `control.write` 权限的 read-only viewer
- `takeoverEligible = true` 当前覆盖：
  - foreign stale claim
  - foreign legacy `untracked` claim
- takeover / claim / release 的状态机语义本轮未变；本轮只是把既有 allow/deny 语义显式投影到 inventory surface

## 3. 明确边界
- 本轮没有：
  - 治理确认后的 legacy `untracked` allow/deny policy
  - configurable lease window / SLA
  - timeout reclaim / automatic stale detection
  - stale-aware bulk `repair-shared-channel-sync`
  - release-ready exactly-once delivery semantics
- 当前仍是 `control-plane-api` 内部的 operator-visible eligibility seam，而不是完整的 lease governance engine

## 4. Fresh Evidence
- red：
  - `cargo test -p control-plane-api --offline --test social_external_collaboration_test test_control_plane_social_shared_channel_pending_release_returns_claim_to_unowned_pool -- --nocapture`
  - `cargo test -p control-plane-api --offline --test social_external_collaboration_test test_control_plane_social_shared_channel_pending_takeover_transfers_claim_to_foreign_operator -- --nocapture`
  - 初始失败点均为 `takeoverEligible = null`
- green：
  - release lifecycle 断言 read-only inventory 视角下 `takeoverEligible = false -> false -> false`
  - takeover lifecycle 断言 foreign writer 视角下 `takeoverEligible = false -> true -> true -> false`
  - `cargo fmt -p control-plane-api` = `passed`
  - `cargo test -p control-plane-api --offline --tests -- --nocapture` = `75 passed`

## 5. 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop69 补齐的是 `pending/dead-letter inventory takeoverEligible visibility seam`，不是 `legacy untracked claim policy` 决策
