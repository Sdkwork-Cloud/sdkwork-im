# v0.0.68 - Loop 68 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“`leaseExpiresAt` 已可见，且 takeover 已尊重 active lease”推进到“operator inventory 直接返回 `leaseStatus`”的最小 contract

## 本轮实现
- `services/control-plane-api/src/lib.rs`
  - 新增 `SocialSharedChannelSyncLeaseStatus`
  - `PendingSharedChannelSyncRequest` 新增 `lease_status(&self, now)` 派生逻辑
  - `SocialSharedChannelSyncInventoryItemResponse` 新增 `leaseStatus`
  - pending/dead-letter inventory 通过共享 response helper 返回 `leaseStatus`
  - legacy claimed item 若缺失 `leaseExpiresAt`，当前显式返回 `untracked`
  - 本轮没有改动 claim / release / takeover 的执行语义
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - release lifecycle 当前新增 `leaseStatus = unclaimed -> active -> unclaimed`
  - takeover lifecycle 当前新增 `leaseStatus = unclaimed -> active -> untracked -> stale -> active`
  - 通过 runtime-dir 回写旧快照，显式验证 legacy `leaseExpiresAt = null` 会落到 `untracked`

- 当前 `repair-shared-channel-sync` 仍不是 stale-aware policy
- 当前没有 release-ready exactly-once semantics
