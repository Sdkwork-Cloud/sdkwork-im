# v0.0.64 - Loop 64 - 2026-04-11

## 本轮目标
- 把 `shared-channel sync` 从“owner claim/release 已存在，但 claim 时间不可见”推进到“inventory 显式暴露 `claimedAt`”的最小可审阅 contract

## 本轮实现
- `services/control-plane-api/Cargo.toml`
  - 复用现有 `im-time`，避免本轮为 claim 时间格式引入新的时间实现
- `services/control-plane-api/src/lib.rs`
  - `PendingSharedChannelSyncRequest` 新增 durable `claimedAt`
  - pending/dead-letter inventory item 显式返回 `claimedAt`
  - owner claim 首次写入 UTC RFC3339 毫秒精度 `claimedAt`
  - 同 owner 重复 claim 保留原 `claimedAt`
  - owner-only targeted release 清空 `ownerActorId / ownerActorKind / claimedAt`
- `services/control-plane-api/tests/social_external_collaboration_test.rs`
  - 把 Loop63 release lifecycle e2e 升级为 claimedAt 可见性回归，验证 `claim 前 null -> claim 后非空 -> release 后 null`

## 风险与未闭环
- 当前还没有 `leaseExpiresAt / SLA` 元数据
- 当前 foreign operator 仍无法对 abandoned claim 做 force release / takeover
- 当前 `repair-shared-channel-sync` 仍是粗粒度全量 operator route，尚未并入 lease policy language
- 当前没有 release-ready exactly-once semantics
- `https://` public runtime target 仍未进入本轮 truth

## 结论
- `S07` 仍是 `not_closed / local_closure`
- Loop64 已把 `shared-channel sync` 补齐到 `durable pending backlog + operator repair + pending inventory + targeted pending republish + pending targeted claim + republish ownership guard + pending targeted release + pending claimedAt visibility + next healthy ready-pair write auto-retry + repeated-failure dead-letter + dead-letter inventory + dead-letter targeted/all requeue + failure-budget rearm`
- 下一主缺口收敛为 `stale-claim takeover / lease / SLA semantics`
